### 3.1 AI 智能推荐页 (`pages/index/index`)

**路径**：`pages/index/index`（TabBar[0]）  
**核心逻辑**：四阶段状态机 — `welcome → qa → loading → result`

#### 阶段一：欢迎引导 (`stage === 'welcome'`)

| 交互部件 | 触发位置 | 当前行为 | 目标行为 | 所需后端 API |
|----------|----------|----------|----------|-------------|
| **「开始探索」按钮** | `index.wxml:41` `bindtap="startQA"` | 切换到问答阶段（纯前端状态切换） | 同当前，无需后端 | 无 |

#### 阶段二：问答流 (`stage === 'qa'`)

问题数据通过 `getRecommendQuestions()` 从后端加载（`GET /recommend/questions`），失败时显示错误提示。支持单选和多选两种题型。

| 交互部件 | 触发位置 | 当前行为 | 目标行为 | 所需后端 API |
|----------|----------|----------|----------|-------------|
| **选项点击（单选/多选）** | `question-card` 组件 `bindanswer="onAnswer"` | 本地 `answers` 对象更新，触发震动反馈 | 同当前，无需后端 | 无 |
| **「下一题」按钮** | `question-card` 组件 `bindnext="nextQuestion"` | 切下一题或调用 `getRecommendations()` 提交答案给后端 AI 引擎 | 同当前 | **`POST /recommend`** — 提交问答答案，返回推荐雪茄列表 |
| **「重试」按钮**（加载失败时） | `index.wxml` `bindtap="retryLoad"` | 重新加载问题列表 | 同当前 | **`GET /recommend/questions`** |

**问题数据结构**（后端需接收）：
```json
{
  "1": [0],         // 场合：单选，选项索引
  "2": [2],         // 强度：单选
  "3": [1, 3, 5],   // 风味：多选
  "4": [2],         // 时长：单选
  "5": [3]          // 经验：单选
}
```

#### 阶段三：加载中 (`stage === 'loading'`)

| 交互部件 | 说明 | 所需后端 API |
|----------|------|-------------|
| `<skeleton-cigar>` × 3 | 骨架屏占位，提示 "AI 正在为您匹配最佳雪茄..." | 无（纯前端动画） |

#### 阶段四：推荐结果 (`stage === 'result'`)

| 交互部件 | 触发位置 | 当前行为 | 目标行为 | 所需后端 API |
|----------|----------|----------|----------|-------------|
| **「详情」按钮** | `index.wxml` `bindtap="viewDetail"` | `wx.navigateTo` 跳转详情页（传 `id`） | 同当前 | 无（纯前端路由） |
| **「加入购物车」按钮** | `index.wxml` `bindtap="addToCart"` | 检查登录态，调用 `addToCart()` API 加入购物车，成功后震动+Toast+更新角标 | 同当前 | **`POST /cart/add`** — 加入购物车 |
| **「重新测试」按钮** | `index.wxml` `bindtap="restart"` | 重置所有状态回到 welcome | 同当前 | 无 |
| **右上角分享** | `onShareAppMessage` | 分享小程序卡片 | 同当前 | 无 |
| **「刷新」按钮**（结果为空时） | `index.wxml` `bindtap="retryLoad"` | 重新加载推荐结果 | 同当前 | 无 |

### 3.3 购物车页 (`pages/cart/cart`)

**路径**：`pages/cart/cart`（TabBar[2]）  
**核心逻辑**：从后端加载购物车列表（`getCart()`），支持乐观更新数量/删除 + 后端同步。区分未登录空态和已登录无商品空态。

| 交互部件 | 触发位置 | 当前行为 | 目标行为 | 所需后端 API |
|----------|----------|----------|----------|-------------|
| **数量减号 `−`** | `cart.wxml` `bindtap="changeQty"` `data-delta="-1"` | 乐观更新本地数量（最少 1），同步 tabBar 徽标，调用 `updateCartQty()` 同步后端 | 同当前 | **`PUT /cart/:id`** — 更新商品数量 |
| **数量加号 `+`** | `cart.wxml` `bindtap="changeQty"` `data-delta="1"` | 乐观更新本地数量，同步后端 | 同当前 | 同上 |
| **删除 `×` 按钮** | `cart.wxml` `bindtap="removeItem"` | 乐观删除，小于等于1时直接移除，调用 `removeCartItem()` 同步后端 | 同当前 | **`DELETE /cart/:id`** — 移除购物车商品 |
| **「去结算」按钮** | `cart.wxml` `bindtap="checkout"` | 检查登录态后跳转 checkout 页（结算页自行校验库存） | 同当前 | 无 |
| **「立即登录」按钮**（未登录空态） | `cart.wxml` `bindtap="doLogin"` | 触发 `wx.login` → `POST /auth/wechat-login`，登录后重新加载购物车 | 同当前 | **`POST /auth/wechat-login`** |
| **「去探索」按钮**（已登录无商品空态） | `cart.wxml` `bindtap="goExplore"` | `wx.switchTab` 切换到 AI 推荐 Tab | 同当前 | 无 |

**购物车数据结构**：

## 七、推荐技术栈

### 7.1 后端主服务

**推荐方案：Node.js + Nest.js**

| 层面 | 推荐选型 | 理由 |
|------|----------|------|
| **运行时** | Node.js 18+ LTS | 与小程序前端同为 JavaScript 生态，团队学习成本低 |
| **框架** | Nest.js | 企业级架构（Module/Controller/Service 分层），内置依赖注入、守卫、拦截器、管道，天然支持 TypeScript |
| **ORM** | Prisma 或 TypeORM | Prisma 类型安全、迁移管理优秀；TypeORM 与 Nest.js 集成更成熟 |
| **数据库** | PostgreSQL 14+ | 双等级体系涉及复杂积分计算和事务操作，需要 ACID 保证；JSON 字段灵活存储风味标签等半结构化数据 |
| **缓存** | Redis | Session/Token 管理、热门雪茄缓存、等级规则缓存、限流 |
| **文件存储** | 阿里云 OSS / 腾讯云 COS | 雪茄图片、海报文件、录音文件存储 |
| **消息队列** | RabbitMQ / Redis Streams | 订单同步美团收银、等级积分异步计算、推送消息异步处理 |

### 7.2 AI 服务（可独立部署）

| 层面 | 推荐选型 | 理由 |
|------|----------|------|
| **推荐引擎** | Python + FastAPI | 雪茄推荐算法（协同过滤/内容匹配）适合 Python ML 生态 |
| **语音识别** | 微信云托管语音识别 / 腾讯云 ASR | 小程序原生能力，无需额外对接 |
| **NLP 风味提取** | Python + jieba 分词 + 规则引擎 | 从语音文本中提取前/中/尾段风味关键词 |

### 7.3 支付对接

| 支付方式 | 对接方案 |
|----------|----------|
| **微信支付（充值）** | 微信支付 API v3（JSAPI 支付）+ 回调通知 |
| **美团收银** | 美团收银开放平台 API（订单同步 + 支付状态回调） |

### 7.4 基础设施

| 层面 | 推荐选型 | 理由 |
|------|----------|------|
| **部署** | Docker + K8s / 云托管 | 微信云托管可直接部署容器，免运维 |
| **网关** | Nginx / Kong / 微信云托管网关 | 统一鉴权、限流、日志 |
| **监控** | Prometheus + Grafana / 腾讯云监控 | 接口耗时、错误率、订单量监控 |
| **日志** | ELK / 腾讯云 CLS | 集中日志分析 |

## 八、前后端数据交互规范建议

### 8.1 请求格式

```
Content-Type: application/json
Authorization: Bearer {jwt_token}
```

### 8.2 统一响应体

```json
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "timestamp": 1714400000000
}
```

### 8.3 错误码体系

| code | 含义 |
|------|------|
| `0` | 成功 |
| `1001` | Token 过期，需重新登录 |
| `1002` | Token 无效 |
| `2001` | 参数校验失败 |
| `3001` | 余额不足 |
| `3002` | 库存不足 |
| `3003` | 商品已下架 |
| `4001` | 订单状态不允许此操作 |
| `4002` | 已评价过，不可重复评价 |
| `5001` | 敏感词拦截 |
| `9001` | 服务器内部错误 |

### 8.4 分页规范

```json
// 请求
GET /api/xxx?page=1&size=20

// 响应
{
  "list": [...],
  "page": 1,
  "size": 20,
  "total": 128,
  "totalPages": 7
}
```
