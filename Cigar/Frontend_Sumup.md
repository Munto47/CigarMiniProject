# 雪茄俱乐部微信小程序 — 前端设计总结文档

> 文档目标：让从未看过该项目的人，读完本文后能够完整理解前端的设计逻辑与视觉风格，并立刻提出有针对性的改进方向。

---

## 一、项目概貌

| 项目属性 | 内容 |
|---|---|
| **应用类型** | 微信小程序（纯前端，无后端） |
| **渲染引擎** | Skyline（微信新一代高性能渲染引擎）|
| **组件框架** | Glass-Easel（对应 Skyline 的新组件框架）|
| **代码加载** | lazyCodeLoading: requiredComponents（按需加载）|
| **设计定位** | 高端奢侈品品鉴应用，对标国际顶级雪茄俱乐部 |
| **核心色调** | 黑金——深黑底色 + 金色主色调 |
| **页面总数** | 8 个页面、7 个可复用组件 |

**产品定位一句话**：一个面向高净值人群的雪茄电商 + AI 品鉴工具，线下俱乐部的数字化延伸。

## 二、视觉设计系统（Design System）

这是整个小程序视觉统一性的核心，所有页面和组件均基于此系统搭建。

### 2.1 颜色体系

```
主色（金）
  --color-gold:        #C9A84C   ← 主要强调色，所有 CTA、标题、图标
  --color-gold-light:  #E8C97A   ← 高亮状态、渐变起始
  --color-gold-dim:    #7A6430   ← 渐变末端、次要金色

背景层级（三层黑）
  --bg-base:           #0D0D0D   ← 页面最底层背景（近纯黑）
  --bg-surface:        #161616   ← 卡片背景
  --bg-elevated:       #1F1F1F   ← 弹层/悬浮元素背景
  --bg-overlay:        rgba(0,0,0,0.72)  ← 遮罩层

文本层级（三档）
  --text-primary:      #F5F0E8   ← 主要文字（米白，非纯白，符合奢侈品调性）
  --text-secondary:    #9E9484   ← 次要文字/说明文字
  --text-disabled:     #4A4540   ← 禁用/占位文字

边框（金色半透明）
  --border-subtle:     rgba(201,168,76,0.15)  ← 低强度边框（卡片）
  --border-strong:     rgba(201,168,76,0.45)  ← 高强度边框（ghost 按钮）

功能色
  --color-success:     #4CAF7A   ← 成功/已购标签
  --color-warning:     #E8A04C   ← 警告
  --color-danger:      #C94C4C   ← 删除/错误
```

**视觉效果**：整体页面呈现「沉浸式暗黑 + 金色点睛」的质感，类似高端夜店或品酒室的氛围。金色边框和文字在深黑底色上有明显的呼吸感，不刺眼却富有质感。

### 2.2 间距与圆角

```
间距（rpx）:  xs=8  sm=16  md=24  lg=40  xl=64
圆角（rpx）:  sm=12  md=24  lg=40  full=9999（胶囊形）
```

卡片和容器普遍使用 `radius-md (24rpx)`，按钮使用 `radius-full (胶囊形)`，保持圆润现代的风格。

### 2.3 全局阴影

```
--shadow-gold:  0 4rpx 24rpx rgba(201,168,76,0.18)   ← 金色光晕阴影
--shadow-card:  0 8rpx 40rpx rgba(0,0,0,0.6)          ← 卡片深阴影
```

卡片浮在背景上的立体感来自 `shadow-card`；金色强调元素下方有 `shadow-gold` 的隐约光晕。

### 2.4 全局动画库

| 动画名 | 效果描述 | 典型用途 |
|---|---|---|
| `fadeIn` | 透明度 0→1，600ms | 页面/区块出现 |
| `slideUp` | Y轴 +60rpx→0 + 淡入 | 卡片、结果列表出现 |
| `slideDown` | Y轴 -40rpx→0 + 淡入 | 顶部提示 |
| `shimmer` | 背景渐变横扫（无限循环）| 骨架屏扫光 |
| `pulse` | 金色光圈向外扩散（无限循环）| 录音按钮等候中 |
| `breathe` | 透明度 + 轻微缩放（无限循环）| 雪茄烟雾、装饰元素 |

### 2.5 全局公共组件类

以下 CSS 类在 `app.wxss` 中全局定义，所有页面直接复用：

- **`.card`**：深色卡片容器（`#161616` 背景 + 金色细边框 + 深阴影）
- **`.gold-btn`**：金色渐变主按钮（胶囊形，135° 渐变，深色文字）
- **`.ghost-btn`**：幽灵次要按钮（透明背景 + 金色边框 + 金色文字）
- **`.section-title`**：区块标题（金色，700 字重，0.1em 字间距）
- **`.chip`**：风味/属性标签（金色半透明背景，胶囊形）
- **`.rating-badge`**：评分徽章（金色文字，半透明金色背景）

## 三、页面详细说明

### 3.1 首页 / AI 推荐（pages/index）

**TabBar 入口**：AI 推荐（第1个）

这是整个小程序的核心功能页，分四个串联阶段。

#### 阶段一：欢迎引导（Welcome）

**视觉效果**：
- 页面正中央是一支纯 CSS 绘制的雪茄剪影（圆角矩形 body + 尖端 tip + 腰带 band），上方有 3 条 breathe 动画的烟雾飘散
- 下方是品牌名 `CIGAR CLUB`（英文小标题）+ 「发现属于你的」+「专属雪茄」（其中「专属雪茄」以金色显示）
- 说明文字：「通过 AI 智能分析您的偏好与场景，精准匹配最适合您的雪茄品类」
- 主 CTA：金色渐变胶囊按钮「开始探索」
- 按钮下方有一条「约需 2 分钟」的时间提示（左右各一条短横线，中间文字）

**状态机**：`stage = 'welcome'`

#### 阶段二：5 题问答（QA）

**视觉效果**：
- 顶部金色进度条（`x/5` 进度填充动画）
- 中心区域是 `question-card` 组件（见组件章节），卡片切换时有左滑淡出 + 右滑淡入的帧动画（card-exit / card-enter 类名切换，延迟 280ms）
- 选项点击触发 `wx.vibrateShort({ type: 'light' })` 轻触觉反馈

**5 道题目**：
1. 享用场合（单选）：商务会谈 / 独处休闲 / 朋友聚会 / 庆祝特殊场合
2. 强度偏好（单选）：轻柔温和 / 均衡适中 / 浓郁丰厚 / 醇厚强劲
3. 风味特征（**多选**）：果香甜润 / 木质烟草 / 泥土矿物 / 咖啡可可 / 辛香胡椒 / 奶油丝滑
4. 品鉴时长（单选）：30分钟以内 / 30–60分钟 / 1–2小时 / 悠然不限
5. 品鉴经验（单选）：初次尝试 / 偶尔品鉴 / 资深爱好者 / 专业玩家

#### 阶段三：AI 加载（Loading）

**视觉效果**：
- 顶部文字「AI 正在匹配...」（loading-dots 类，三个点可添加动效）
- 下方 3 个 `skeleton-cigar` 骨架屏组件叠放（雪茄轮廓占位 + 3 条文字骨架线），每个都有 shimmer 扫光动画
- 加载时长固定 2200ms（mock 延时）

#### 阶段四：推荐结果（Result）

**视觉效果**：
- 标题区：「为您推荐」+ 副标题
- 3 张雪茄卡片（第1张为 featured 样式，尺寸或样式略有不同）
- 每张卡片包含：
  - 右上角匹配度徽章（如「98% 匹配」）
  - 雪茄图片区（纯 CSS 雪茄图形占位）
  - 雪茄名称 + 评分徽章（`rating-badge`）
  - 产地 · 强度 · 品鉴时长（元数据行）
  - 风味标签（`.chip` 多个）
  - 价格 + 「详情」ghost 按钮 + 「加入购物车」gold 按钮
- 底部「重新测试」ghost 按钮

**当前 Mock 数据**：
| 雪茄 | 产地 | 价格 | 匹配度 |
|---|---|---|---|
| Cohiba Behike 52 | 古巴 | ¥1280 | 98% |
| Davidoff Winston Churchill | 多米尼加 | ¥680 | 94% |
| Arturo Fuente OpusX | 多米尼加 | ¥960 | 91% |

**数据流向**：点击「详情」→ navigateTo 跳转至雪茄详情页（传 id 参数）；点击「加入购物车」→ Toast 提示 + tab-bar 购物车徽标数量 +1

### 3.2 风味生成（pages/flavor）

**TabBar 入口**：风味生成（第2个）

**整体布局**：全页 scroll-view 竖向滚动

**视觉效果**：

1. **Hero 标题区**：
   - 小标「FLAVOR PROFILE」（金色，0.2em 字间距）
   - 主标题「描述你的感受」
   - 副标题「选择风味关键词，AI 为您生成专属图谱」

2. **两张入口卡片**（横向并排）：
   - **语音描述卡**：麦克风图标（纯 CSS 实现）+ 标题 + 描述，点击跳转 poster 页
   - **关键词选择卡**（默认激活态，有选中边框高亮）：标签图标 + 标题 + 描述

3. **风味关键词标签云**（12 个 `pairing-chip` 组件）：
   - 果香甜润 / 木质烟草 / 泥土矿物 / 咖啡可可 / 辛香胡椒 / 奶油丝滑
   - 皮革木桶 / 花香清雅 / 坚果焦糖 / 香草甜美 / 泥炭烟熏 / 雪松丝绸
   - 每个 chip 可独立切换选中/未选中状态，选中触发 vibrateShort

4. **风味图谱区**：
   - 未选择时：显示一个六边形空态占位图 + 「选择风味关键词后展示图谱」提示文字
   - 选择后：显示 `flavor-radar` 组件（雷达图 + 进度条明细）

5. **底部操作区**：
   - 主按钮「生成风味海报」（gold-btn）→ 跳转 poster 页
   - 次要按钮「加入品鉴记录」（ghost-btn）→ Toast 提示
   - 条件显示「重置选择」文字按钮（当 hasResult=true 时）

**数据逻辑**：选中标签后实时计算各维度分值，触发 radar 图谱重绘。具体分值基于标签分类的 mock 计算函数（如「果香甜润」→ 果香分 +30），并非真实 AI 接口。

## 四、组件库详解

### 4.1 navigation-bar（自定义导航栏）

**文件**：`components/navigation-bar/`

**功能**：替代微信原生导航栏，实现 iOS/Android 安全区自适应。

**属性**：
- `title`：导航栏标题
- `back`：是否显示返回按钮（Boolean）
- `color`：文字颜色
- `background`：背景色（支持 `transparent`，配合视差效果使用）

**视觉效果**：主色文字 + 左侧返回箭头（可选）+ 右侧「主页」图标（可选）。背景透明时可叠加在 Hero 图片上，形成沉浸式效果（cigar-detail 页用此方式）。

---

### 4.2 question-card（问答卡片）

**文件**：`components/question-card/`

**功能**：AI 问答流中的问题卡片，支持单选/多选两种类型。

**属性**：

## 六、技术架构与实现特点

### 6.1 纯 CSS 图形绘制（无图片资源）
当前项目**完全没有使用图片文件**，所有视觉元素（雪茄插图、购物袋、麦克风图标、tab 图标等）全部由 CSS 或 Canvas 绘制：
- 雪茄：圆角矩形 body（棕色渐变）+ 细尖 tip（金色）+ 腰带 band
- 购物袋：handle（圆弧）+ body（矩形）
- 波形、图标、装饰圆均为纯 CSS

优点：体积极小，无图片加载延迟。
缺点：图形较为抽象，缺乏真实产品质感。

### 6.2 Canvas 2D 绘制
- **flavor-radar**：getContext('2d') 绘制6轴雷达图，含网格、轴线、数据面、顶点
- **poster**：Canvas 生成整张风味海报（背景渐变 + 文字排版 + 图形元素），支持 wx.canvasToTempFilePath 导出保存

### 6.3 状态管理
- **globalData**：`selectedTab`（当前 tab）、`cartCount`（购物车数量）存于 `app.js`
- **页面级状态**：各页面独立管理，通过 `setData` 驱动视图更新
- **跨页通信**：tab-bar `setData({ cartCount })` 直接更新徽标；购物车数据目前不持久化

### 6.4 触觉反馈
全局统一使用 `wx.vibrateShort({ type: 'light' })` 在以下场景触发：
- 问答选项选择
- pairing-chip 标签切换
- 加入购物车

### 6.5 骨架屏与加载态
- AI 推荐：3个 skeleton-cigar 组件 + 2200ms 固定 mock 延时
- 海报生成：saving 状态按钮变为「保存中...」
- 支付确认：paying 状态按钮变为「处理中...」

## 七、当前限制与改进方向

> 以下是基于现有实现可以直接看出的痛点和优化空间，供改进时参考。

### 7.1 视觉与交互层面

| 问题 | 描述 | 改进建议 |
|---|---|---|
| **无真实产品图** | 所有雪茄均为 CSS 抽象图形，缺乏高端奢侈品的真实质感 | 接入产品图片（雪茄实物/包装盒高清图），替换 CSS 占位 |
| **推荐结果缺乏差异化** | 3张结果卡样式几乎相同，「featured」首位卡片无明显视觉突出 | 首位卡片做大卡（全宽），下面两张做小卡横排，或加金色特效边框 |
| **风味标签云布局单一** | 12个 chip 等宽换行，缺乏轻重感 | 改为不同大小的标签云（高分值标签更大）或分类分组展示 |
| **会员卡缺少卡片感** | 会员卡虽有装饰元素，但背景是纯色+圆形装饰，高端感不足 | 加入金属质感渐变、磨砂纹理、holographic 光效，或做实物卡片翻转动效 |
| **空态设计过于简单** | 购物车空态 CSS 购物袋图示非常简单 | 优化为精致的品牌插图或动效 |
| **历史记录空态缺失** | 没有空记录列表时的空态页面 | 增加空态设计 |
| **骨架屏数量固定** | 始终显示3个骨架，与实际返回数量无关 | 根据预期结果数量动态渲染 |
| **配饮图标全部一样** | 配饮推荐的图标全是同一个圆点占位 | 区分威士忌/咖啡/雪松水等配饮的专属图标 |

### 7.2 功能层面

| 问题 | 描述 | 改进建议 |
|---|---|---|
| **购物车数据不持久** | 刷新/切换 tab 后购物车数据丢失（仅 mock） | 接入 wx.setStorage 本地持久化，或后端接口 |
| **收藏功能缺失** | 历史记录有「收藏」筛选 tab 但无收藏按钮 | 在结果卡和详情页增加收藏心形按钮 |
| **搜索功能缺失** | 没有任何搜索入口 | 在详情或历史记录页增加搜索功能 |
| **问卷不可回退** | 问答过程中只能前进，无法回到上一题 | 增加「上一题」按钮 |
| **推荐结果无法刷新** | 每次必须重新做完5道题才能获得新推荐 | 结果页增加「换一批」快捷刷新 |
| **海报录音无 mock 文字** | 录音完成后显示的 transcript 文字仅为固定 mock | 待接入语音转文字 API |
| **支付流程断层** | 选择美团收银后无实际跳转（仅 Toast）| 待接入实际支付 SDK |
| **品鉴预约无页面** | 会员中心「品鉴预约」点击无跳转 | 需新增预约页面 |

### 7.3 工程层面

| 问题 | 描述 | 改进建议 |
|---|---|---|
| **全部 mock 数据硬编码** | 雪茄数据、用户信息、历史记录均写在 JS 文件常量中 | 提取为独立 mock 数据层，后续无缝替换接口调用 |
| **无全局 loading 状态** | 页面间跳转、加入购物车无统一 loading 组件 | 封装全局 toast/loading 工具函数 |
| **筛选 tab 无数据过滤逻辑** | 历史记录的4个筛选 tab 点击后列表内容无实际变化 | 实现基于 type 字段的过滤逻辑 |
| **雪茄详情 id 未做匹配** | 从推荐列表跳转详情时传入了 id，但详情页未根据 id 动态加载对应数据 | 接入 id→数据 映射逻辑 |
