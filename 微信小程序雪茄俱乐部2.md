微信小程序雪茄俱乐部2.0需求文档

一、文档概述

1.1 文档目的

本文档用于明确雪茄俱乐部微信小程序的开发需求，梳理核心功能、业务逻辑、交互规则、后台管理及技术对接等相关要求，为开发人员提供清晰的开发依据，为产品测试提供明确的验收标准，为商家运营提供参考指南，确保小程序开发符合商家预期，满足用户使用需求，实现线上服务与线下运营的高效联动。

1.2 项目背景

商家运营线下雪茄俱乐部，核心客群为雪茄爱好者，现有线下服务模式难以满足用户个性化推荐、便捷下单及社交分享需求。 为拓展线上服务渠道，提升用户体验，增强用户粘性，实现用户数据化管理、品牌精准曝光及订单高效处理，特开发本微信小程序，聚焦智能雪茄推荐、风味海报生成、线上下单结算三大核心场景，联动美团收银系统及后台管理端，打造 “线上服务 + 线下体验” 的一体化雪茄服务载体。

1.3 核心目标

- 实现 AI 智能雪茄推荐功能，根据用户选择的问题匹配对应雪茄，并推送适配配饮，满足用户个性化选购需求；
- 实现雪茄风味生成器功能，支持用户语音描述风味，生成带商家 LOGO 的专属海报，支持保存、转发，助力品牌传播；
- 实现用户数据存储，记录用户历史选择、订单及海报生成记录，方便用户回顾、商家分析；
- 打造简洁、易用的交互界面，贴合雪茄俱乐部的高端调性，提升用户使用体验及留存率；
- 实现小程序下单功能，支持用户直接选购雪茄、饮品，接入美团收银系统，实现下单数据实时同步及高效结算；
- 搭建后台管理端，支持商家对商品、订单、用户、系统参数进行全面管理，提升运营效率；
- 实现线上线下数据联动，确保小程序、后台管理端、美团收银系统数据一致，避免数据偏差；
- 新增会员储值模块，支持微信支付充值、储值折扣、会员生日提醒、商品上新提醒；
- 新增 AI 雪茄风味解析 ，展示本店雪茄完整风味，并推荐相似口味（本店在售 + 行业参考）；
- 支持 雪茄数据库动态扩容 ，可随时新增品牌 / 型号，支持批量维护、实时同步；
- 支付方式仅支持会员储值支付或跳转美团收银系统支付。
- **新增雪茄真实评分与评论体系，支持订单完成后弹窗快捷评价，沉淀用户真实反馈以辅助社区互动及选购参考。**

二、核心需求梳理

2.1 整体定位

本微信小程序是雪茄俱乐部的线上服务入口，聚焦 “AI 智能雪茄推荐”“风味海报生成”“会员储值消费”“线上下单结算”“ AI 风味解析与相似推荐 ” 五大核心功能，**并辅以“商品评分与评论”增强社区互动**，兼顾用户数据管理、品牌展示及后台运营管理，面向雪茄爱好者，提供个性化、便捷、高端的雪茄相关服务，同时为商家提供高效的运营管理工具，实现线上引流、线下转化的闭环。

2.2 核心功能需求

2.2.1 AI 智能雪茄推荐模块（原智能雪茄推荐模块升级）

- 用户交互：通过简洁的问答形式引导用户选择，问题聚焦口味偏好、抽吸习惯、预算、场景等核心筛选维度，用户完成选择后，AI 系统自动分析并匹配对应雪茄，支持返回修改答案；
- 雪茄推荐规则：根据用户问答结果，AI 算法多维权重匹配，从系统雪茄库中筛选适配的雪茄，自动推荐 3 款最优匹配选项，按匹配度从高到低排序，匹配不足 3 支时展示全部匹配结果并提示用户调整选项；
- 雪茄信息展示：每款推荐雪茄需完整展示核心信息，包括：雪茄图片、名称、风味描述（前段、中段、尾段）、抽吸时长、原价、会员储值价、规格（单支 / 礼盒）、库存状态；
- 配饮推送：根据每款推荐雪茄的口味特点，自动匹配并推送 1-2 款适配配饮（如威士忌、鸡尾酒等），标注配饮名称、适配理由及参考价格，支持配饮直接加入购物车；
- 用户数据存储：记录用户每次选择的问答结果、推荐的雪茄信息，保存用户历史选择记录，支持用户快速查看上一次所选的雪茄及相关信息（风味、抽吸时长、价格、配饮）；
- 新增：相似雪茄推荐 ：基于当前推荐雪茄风味，AI 在本店数据库匹配同风味 / 同浓郁度相似款，标注相似度，支持直接加购。

2.2.2 风味生成器模块

- 语音输入：支持用户通过语音描述的方式，录入单支雪茄的前段风味、中段风味、尾段风味，语音识别支持普通话，兼容常见口语表达，录音时长控制在 10-30 秒，支持暂停、重新录音；
- 风味识别与整理：系统调用微信语音识别接口，将用户语音转换为文字，自动提取前段、中段、尾段的风味关键词，整理成规范的风味描述，允许用户手动修改、补充、删除，确保风味描述准确；
- 海报生成：根据用户录入的风味信息，生成专属风味海报，海报需包含：商家固定 LOGO（位置固定、清晰可见）、雪茄风味详情（分段标注）、核心标题、俱乐部名称及小程序入口标识，设计风格贴合雪茄高端调性，简洁大气；
- 海报操作：生成海报后，支持用户一键保存海报至手机相册，支持一键转发至微信朋友圈、微信好友，转发时自动携带小程序入口，便于品牌传播；
- 海报历史：用户生成的所有风味海报均保存至历史记录，支持用户重新查看、保存、转发；
- 新增：快捷导入风味 ：支持从本店雪茄库一键导入已解析的标准风味描述，快速生成海报。

2.2.3 会员储值模块

- 会员开通：用户微信授权登录后自动成为普通会员，无需额外申请；
- 储值充值：支持固定档位充值与自定义金额充值，充值仅支持微信支付，充值成功后余额实时到账；
- 储值余额：仅限小程序内消费使用，不提现、不退款、不找零；
- 储值折扣：储值会员享受统一折扣，下单时自动按折扣计算金额；
- 生日管理：会员可填写 / 修改生日信息，系统自动进行生日提醒，并提供生日专属权益；
- 上新提醒：后台商品上新时，系统自动向会员推送上新通知，新品标注 “NEW” 标识；
- 会员中心：展示会员等级、储值余额、折扣率、生日信息、充值入口、消费明细、充值明细。

2.2.4 用户数据管理模块

- 数据存储范围：用户微信授权信息（昵称、头像）、会员信息、储值余额、充值记录、消费记录、问答记录、历史选择的雪茄信息、风味海报生成记录、订单记录（待支付、待结算、已完成、已取消） **、雪茄评分与评论记录**；
- 数据查看：用户可在小程序 “历史记录”“我的订单”“会员中心” 页面，查看自己的历史选择、海报生成记录、订单详情、储值明细，支持按时间倒序排序；
- 数据管理：用户可手动删除自己的历史选择、海报记录，可取消未支付、未结算的订单；
- 数据安全性：用户数据与微信账号绑定，仅本人可见，严格保护用户隐私，不泄露用户个人偏好、订单信息，符合微信小程序隐私政策要求。

2.2.5 下单与收银对接模块

- 商品选购：雪茄详情页、推荐列表页设置 “加入购物车”“立即购买” 按钮，单独设置 “饮品选购” 入口，支持用户选择商品规格、数量，查看实时库存、原价与会员储值价；
- 购物车管理：支持用户查看购物车商品，调整商品数量、删除商品，支持勾选商品结算，显示商品总价、储值折扣金额、实付金额；
- 订单提交：用户提交订单前，确认收货方式为到店自提、自提时间，提交后生成唯一订单号，实时推送至美团收银系统；
- 支付方式（仅支持两种）：
    - 会员储值余额支付；
    - 跳转美团收银系统支付；
- 收银同步：小程序订单数据（订单号、商品信息、金额、用户信息、收货方式、支付方式）实时同步至美团收银系统，商家可通过收银系统接收订单、核对信息、完成结算，订单状态（待支付、待结算、已完成、已取消）在小程序与收银系统同步更新。

2.2.6 AI 雪茄风味解析模块

- 风味 AI 解析 ：用户进入任意本店雪茄详情页，点击【AI 解析风味】，AI 自动读取数据库并结构化展示：前段 / 中段 / 尾段、香气标签、浓郁度、口感特征、抽吸建议、适配场景；
- 风味图谱展示 ：以可视化卡片呈现果香 / 木香 / 坚果 / 巧克力 / 皮革 / 花香等占比，清晰直观；
- 本店相似雪茄推荐 ：基于当前雪茄风味，AI 匹配店内同风味、同浓郁度、同价位相似款，标注相似度（如 92%），支持直接加入购物车；
- 行业风味参考推荐 ：展示非本店在售、但风味接近的行业品牌 / 型号，仅作风味参考，不展示价格与购买入口，标注 “非本店在售”；
- 风味一键复制 ：支持复制标准风味描述，快速跳转至风味生成器生成海报。

2.2.7 后台管理端模块

- 账号管理：支持商家创建多个管理账号，分配不同权限（超级管理员、商品管理员、订单管理员等），支持密码修改、账号禁用 / 启用；
- 商品管理：支持添加、修改、删除雪茄、饮品商品信息，同步更新至小程序前端及美团收银系统，支持商品分类管理；
- 雪茄库管理（升级） ：
    - 双库管理： 本店在售雪茄库 （销售 / 推荐 / 下单）+ 行业风味参考库 （非卖 / 风味参考）；
    - 单条新增 / 编辑：录入完整字段后一键同步小程序、AI 引擎、收银系统；
    - 批量维护：支持 Excel 批量导入 / 导出，自动校验必填项、重复数据；
    - 标签动态配置：新增 / 编辑 / 删除风味标签，AI 推荐规则实时生效；
    - AI 特征库刷新：数据变更后 AI 实时更新，新雪茄立即进入推荐池；
- 订单管理：查看所有用户提交的订单，同步美团收银系统的结算状态，可手动处理异常订单，支持筛选与导出 Excel；
- 用户 / 会员管理：查看用户 / 会员列表与详情，支持按活跃度、消费金额筛选，用于运营统计；
- 会员储值管理：配置充值档位、储值折扣、生日 / 上新提醒规则，查看流水，支持手动调整余额；
- 系统设置：配置商家 LOGO、小程序信息、美团收银接口、支付参数，查看系统日志；
- 数据统计：查看商品销量、订单量、用户活跃度、储值数据等，生成报表导出；
- 海报管理：查看用户海报记录，配置海报模板、LOGO、色调、字体。
- **评价管理（新增）：支持查看商品评分数据，集中管理用户雪茄评论，支持敏感词过滤及人工删除违规评论功能。**

2.2.8 基础功能

- 小程序登录：支持微信授权一键登录，无需额外注册，首次登录自动获取用户微信昵称、头像，后续自动登录；
- 页面导航：清晰的底部导航栏设计，包含 “AI 智能推荐”“风味生成”“购物车”“历史记录”“会员中心 / 俱乐部”5 个核心入口；
- 品牌展示：会员中心展示俱乐部简介、联系方式、地址、营业时间、环境图，启动页展示 LOGO / 宣传图（3-5 秒）；
- 消息推送：支持会员生日提醒、商品上新提醒、储值到账 / 消费通知、订单状态通知；
- 异常处理：针对语音识别失败、网络异常、数据加载失败、无匹配雪茄等场景，给出清晰提示。

**2.2.9 评价与评分模块（新增）**

- **雪茄评分**：基于5分制雪茄图标进行可视化打分，前端直观展示评分总览。
- **雪茄评论**：详情页独立留言板模式，支持文字评论，供用户交流选购心得。
- **订单后置评价**：订单流转完成后自动触发弹窗闭环，快捷收集真实消费反馈。

三、详细功能说明

3.1 AI 智能雪茄推荐模块详细说明

**3.1.1 问答环节设计**

- 问答数量：控制在 5-8 个问题，避免过多占用用户时间；
- 问题维度（固定 + 可配置）：口味偏好、抽吸时长、预算、风味倾向、抽吸场景、雪茄类型、抽吸经验；
- 交互方式：单选为主，风味倾向支持多选，支持返回修改，提交后 AI 立即生成推荐。

**3.1.2 AI 推荐逻辑**

- 系统预设雪茄库，每支雪茄录入完整信息与 AI 标签；
- 用户提交答案后，AI 计算匹配权重，高精度匹配雪茄库；
- 推荐匹配度最高的 3 支，按匹配度排序，不足 3 支展示全部并提示；
- 支持重新匹配，修改答案后重新生成列表。

**3.1.3 雪茄及配饮展示**

- 单支雪茄详情页：展示高清图片、名称、规格、价格、库存、抽吸时长、分段风味、适配配饮，底部设加入购物车 / 立即购买； **新增：详情页同步展示该商品的综合平均分与评价总人数，并内嵌独立评论模块供用户查阅历史留言。**
- 推荐列表页：3 支横向排列，展示缩略图、名称、核心风味、价格、匹配度，点击进详情； **新增：列表商品信息中直观展示商品评分状况。**
- 配饮展示：单独入口，按品类分类，支持搜索筛选，可加购。

**3.1.4 历史记录查看**

- 历史记录分 “雪茄选择记录”“海报生成记录”，按时间倒序；
- 突出上一次选择，展示关键信息，点击可重看详情；
- 支持删除单条记录。

3.2 风味生成器模块详细说明

**3.2.1 语音输入功能**

- 支持语音输入与手动输入双模式，录音时长 10-30 秒；
- 调用微信语音识别，成功率不低于 95%，自动拆分三段风味；
- 支持手动修改、补充、删除风味文字。

**3.2.2 海报生成规范**

- 尺寸 1:1 正方形，清晰度≥720\*720px；
- 包含 LOGO、风味详情、标题、俱乐部名称、小程序入口；
- 风格高端大气，色调、字体、背景可后台配置；
- 生成响应≤3 秒，失败可重试。

**3.2.3 海报操作功能**

- 一键保存至相册，一键转发朋友圈 / 好友，携带小程序入口；
- 支持重新生成，历史记录可再次预览、保存、转发。

3.3 AI 雪茄风味解析模块详细说明（新增）

**3.3.1 功能入口**

- 雪茄详情页 → 【AI 解析风味】按钮

**3.3.2 风味解析展示**

- AI 自动读取数据库，结构化输出：前段 / 中段 / 尾段风味、香气标签、浓郁度、口感特征、抽吸时长、适配配饮 / 场景；
- 可视化风味图谱：展示各香型占比，直观易懂；
- 支持 一键复制风味描述 ，跳转风味生成器快速生成海报。

**3.3.3 相似雪茄推荐**

- 本店相似款 ：同风味 / 同浓郁度 / 同价位，标注相似度，支持加购；
- 行业参考款 ：非本店在售，仅展示品牌、型号、风味，无购买入口，标注 “仅供风味参考”。

**3.3.4 数据联动**

- 解析数据与本店雪茄库实时同步，库内新增 / 修改雪茄，解析内容自动更新。

3.4 会员储值模块详细说明

**3.4.1 会员体系**

- 普通会员：微信授权登录自动开通；
- 储值会员：完成储值自动升级，享受折扣、生日 / 上新提醒。

**3.4.2 储值充值**

- 支持固定档位 + 自定义金额，仅微信支付；
- 余额实时到账，小程序 + 服务通知双重提醒；
- 余额不提现、不退款、不找零。

**3.4.3 储值折扣**

**3.10 评价与评分模块详细说明（新增）**

**3.10.1 雪茄评分功能**

- **评分样式**：以**五个雪茄图标**作为打分基础视觉样式（5个雪茄即为满分），交互上支持按 1-5 分的整数档位进行点选评分。
- **权限控制**：仅限已登录用户可进行评价打分操作，游客身份仅允许查看相关评分结果。
- **限制规则**：针对同一款雪茄，单一用户仅具备一次评价权限。评价一旦提交，不支持重复打分或修改已评定分数，确保数据公正真实。
- **数据展示**：商品列表页及雪茄详情页顶部区域，需直观展示该款雪茄的**综合平均分**以及**评价总人数**。
- **前端交互**：在评分操作区，需通过UI视觉清晰区分“未选中”与“已选中”的雪茄图标状态，直观传达当前选定的评分等级。
- **后端计算**：后端系统接收数据后需自动累加，实时计算最新的平均分并同步至前端各展示页面。

**3.10.2 雪茄评论功能**

- **独立模块**：在雪茄详情页（如详情介绍下方或底部Tab）规划独立的评论版块，集中呈现所有购买/体验用户的相关评论信息。
- **权限控制**：所有访问用户均可浏览评论内容，但仅赋予已登录用户发表“文字内容”评论的权限。
- **安全合规**：用户在提交评论时，后台需接入并经过敏感词/违禁词过滤库筛查，若含违规字眼则拦截禁止发布并提示用户。
- **排序与呈现**：评论列表默认按照发布的时间先后作**倒序展示**（最新评论置顶）。单条评论数据卡片需展示用户昵称、发布时间戳以及具体评论文字内容。
- **永久留存**：所有合规评论数据将作永久化云端存储，保障商品详情页在任何周期内均可正常下发并加载历史评论数据。
- **后台管理**：支持后台管理员对前端所有雪茄评论执行日常筛查及管理，对于漏网的违规或恶意评论可手动执行删除处理。

**3.10.3 订单完成评价弹窗功能**

- **触发时机**：当用户在小程序内下单成功、且系统判定订单流转至“完成”状态后，前端页面自动唤起评价弹窗（或导向独立的评价承接页面）。
- **极简整合**：弹窗界面内嵌上述【雪茄评分组件】+【文字评论输入框】，鼓励用户一站式、快速完成消费后的真实评价。
- **精准评价**：弹窗内容仅针对当前订单内所含的“雪茄类商品”发起的定向评价请求，排除不相关或非雪茄类商品。
- **用户友好**：为防打扰，弹窗强制配置显著的「暂不评价」或「关闭」按钮；评价动作非强制性指标，将选择权交予用户。
- **频次控制**：对于该笔订单涵盖的商品，一旦用户已完成评分与评论流程，系统将其标记，后续不再触发及弹出该评价窗口。
- **平滑流转**：当评价成功提交后，系统给出 Toast 提示并自动销毁弹窗，无缝返回用户原本所在的订单详情页或个人中心。
- **数据追溯**：评价产生的数据需进行底层结构关联，准确绑定对应订单编号、用户UID及商品SKU，以便后台数据穿透及后续溯源。
