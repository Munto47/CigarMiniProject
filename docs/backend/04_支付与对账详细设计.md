## 1. 支付链路总览

### 1.1 三种支付通道

| 通道 | 用途 | 资金流向 | 触发场景 |
|---|---|---|---|
| **微信支付 v3** | 储值充值 | 用户 → 商户 → 系统加余额 | `POST /api/member/recharge` |
| **余额支付** | 线上点单（小程序下单） | 系统扣余额 | `POST /api/orders/{id}/pay` |
| **美团收银** | 线下到店点单 | 用户在收银台扫码 → 美团结算 → 系统记账 | `POST /api/admin/orders` 标 `pay_method=meituan` 等回调 |

### 1.2 数据表关系

```
recharge_orders ─── 1:1 ── payment_records ─── n:1 ── payment_callbacks
       │
       └── 1:n ── balance_transactions / point_transactions / level_change_logs

orders ─── 1:1 ── payment_records ─── n:1 ── payment_callbacks
   │
   ├── 1:n ── order_items
   ├── 1:n ── refund_records
   └── 1:n ── balance_transactions / point_transactions / level_change_logs
```

### 1.3 核心不变量（每日对账校验）

```
∀ user:
  member_profiles.balance_cents == Σ(balance_transactions.amount_cents × direction)

∀ paid order:
  orders.actual_pay_cents == Σ(payment_records.amount_cents WHERE status='success')
  orders.refunded_amount_cents == Σ(refund_records.amount_cents WHERE status='success')
  orders.refunded_amount_cents <= orders.actual_pay_cents

∀ recharge:
  recharge_orders[status='success'].amount_cents == 对应 balance_transactions.amount_cents
```

任一不变量失败 → 立即电话告警（详见 §7.4）。

## 2. 微信支付 v3 接入

### 2.1 申请与配置

商户号（mchid）申请通过后，需要：

| 资料 | 来源 | 配置位置 |
|---|---|---|
| 商户号 mchid | 微信支付商户平台 | `WECHATPAY_MCHID` |
| API v3 密钥 | 商户平台 → 账户中心 → API 安全 | `WECHATPAY_API_V3_KEY`（生产 KMS） |
| 商户私钥 `apiclient_key.pem` | 商户平台下载证书 | 文件挂载（K8s secret） |
| 商户证书序列号 | 私钥配套 | `WECHATPAY_MCH_SERIAL` |
| 微信平台公钥（动态） | 启动时调 `GET /v3/certificates` 拉取 | 内存缓存，每 12h 刷新 |
| 回调 URL | 商户平台配置 | 必须 HTTPS，且备案域名 |

**关键决策**：v3 用 RSA-PKCS1 v1.5 + AES-256-GCM。**不要用 v2**（HMAC-MD5 已过时）。

### 2.2 下单（创建预支付）

```ts
// src/payment/wechat/wechat.service.ts
async createPrepay(params: {
  outTradeNo: string;       // R20260430000001 充值单号
  totalCents: bigint;
  description: string;      // "雪茄俱乐部充值 100 元"
  openid: string;
  notifyUrl: string;        // https://your.domain/api/payment/wechat-callback
}): Promise<{ prepayId: string; payParams: WxPayParams }> {
  const body = {
    appid: this.config.WECHAT_APPID,
    mchid: this.config.WECHATPAY_MCHID,
    description: params.description,
    out_trade_no: params.outTradeNo,
    notify_url: params.notifyUrl,
    amount: { total: Number(params.totalCents), currency: 'CNY' },
    payer: { openid: params.openid },
  };

  const url = 'https://api.mch.weixin.qq.com/v3/pay/transactions/jsapi';
  const auth = await this.buildAuthHeader('POST', '/v3/pay/transactions/jsapi', body);

  const resp = await this.http.post(url, body, {
    headers: { Authorization: auth, 'Content-Type': 'application/json', Accept: 'application/json' },
  });

  if (resp.status !== 200) {
    this.logger.error('wechat prepay failed', { status: resp.status, data: resp.data });
    throw new BusinessException(6001, '微信支付下单失败');
  }

  const prepayId = resp.data.prepay_id;

  // 给小程序前端的二次签名参数（前端 wx.requestPayment 用）
  const payParams = await this.buildPayParams(prepayId);
  return { prepayId, payParams };
}
```

`buildAuthHeader`（v3 签名）：

```ts
private async buildAuthHeader(method: string, urlPath: string, body: any): Promise<string> {
  const timestamp = Math.floor(Date.now() / 1000);
  const nonceStr = randomBytes(16).toString('hex');
  const bodyStr = body ? JSON.stringify(body) : '';

  // 签名串：HTTP方法\nURL路径\n时间戳\n随机串\n请求报文主体\n
  const signStr = `${method}\n${urlPath}\n${timestamp}\n${nonceStr}\n${bodyStr}\n`;
  const signature = createSign('RSA-SHA256').update(signStr).sign(this.privateKey, 'base64');

  return `WECHATPAY2-SHA256-RSA2048 ` +
    `mchid="${this.config.WECHATPAY_MCHID}",` +
    `nonce_str="${nonceStr}",` +
    `signature="${signature}",` +
    `timestamp="${timestamp}",` +
    `serial_no="${this.config.WECHATPAY_MCH_SERIAL}"`;
}
```

### 2.3 回调验签（关键）

```ts
// src/payment/wechat/wechat.verifier.ts
async verify(headers: Record<string, string>, rawBody: Buffer): Promise<boolean> {
  const timestamp = headers['wechatpay-timestamp'];
  const nonce = headers['wechatpay-nonce'];
  const signature = headers['wechatpay-signature'];
  const serial = headers['wechatpay-serial'];   // 微信平台证书序列号

  // 防重放：5 分钟内
  if (Math.abs(Date.now() / 1000 - Number(timestamp)) > 300) {
    this.logger.warn('callback timestamp out of range', { timestamp });
    return false;
  }

  const signStr = `${timestamp}\n${nonce}\n${rawBody.toString()}\n`;

  // 拿到对应 serial 的微信平台证书公钥
  const publicKey = await this.certManager.getPublicKey(serial);
  if (!publicKey) {
    this.logger.error('unknown wechatpay serial', { serial });
    return false;
  }

  return createVerify('RSA-SHA256').update(signStr).verify(publicKey, signature, 'base64');
}
```

**注意点**：
- `rawBody` 必须是**未解析的原始 Buffer**。在 NestJS 用 `app.use(json({ verify: (req, _res, buf) => { (req as any).rawBody = buf; } }))`。
- 微信平台证书定时刷新（12h），缓存所有 serial → publicKey。
- 验签失败立即返回 401，不要进业务事务。

### 2.4 解密 ciphertext

```ts
async decrypt(resource: { ciphertext: string; nonce: string; associated_data: string }): Promise<any> {
  const apiKey = await this.kms.getApiV3Key();   // 32 字节
  const decipher = createDecipheriv('aes-256-gcm', apiKey, resource.nonce);
  const cipher = Buffer.from(resource.ciphertext, 'base64');
  const ciphertext = cipher.subarray(0, cipher.length - 16);
  const authTag = cipher.subarray(cipher.length - 16);

  decipher.setAAD(Buffer.from(resource.associated_data));
  decipher.setAuthTag(authTag);

  const decrypted = Buffer.concat([decipher.update(ciphertext), decipher.final()]);
  return JSON.parse(decrypted.toString('utf8'));
}
```

### 2.5 回调处理完整代码

见 `01_项目结构与编码规范.md §3.4`。要点：
- 验签失败 → 401，**不入业务**
- ON CONFLICT 计数让运维能看到重放
- `payment_callbacks.processed=TRUE` 是第二道幂等
- 业务事务内 `recharge_orders FOR UPDATE` + 状态判断是第三道

### 2.6 错误码映射

| 微信返回 | 我方处理 | 业务码 |
|---|---|---|
| 200 + `SUCCESS` | 正常入账 | 0 |
| 200 + `USERPAYING` | 用户支付中（轮询） | 6002（前端轮询查询） |
| 4xx + `INVALID_REQUEST` | 参数错 | 6003 |
| 5xx | 微信侧异常 | 6001 + 重试 |
| 验签失败 | 不入账 | HTTP 401 |

## 3. 美团收银对接

### 3.1 业务流程

```
店员后台开单（pay_method='meituan'）
   │
   ├─→ 系统调美团下单 API（含金额、商品、商户单号 = orders.order_no）
   │   美团返回 mt_trade_no
   │
   ├─→ 用户/店员在美团收银台支付
   │
美团 → POST /api/payment/meituan-callback
   │
   ├─→ 验签
   ├─→ 写 payment_callbacks
   └─→ 业务事务:
         SELECT orders FOR UPDATE NOWAIT
         幂等命中已 paid → 直接 200
         UPDATE orders status='paid', actual_pay_cents=...
         INSERT payment_records (success, channel='meituan')
         扣库存（stock - locked，locked - qty）
         加积分（消费等级）
         可能 INSERT level_change_logs
```

### 3.2 关键差异

| 维度 | 微信支付 | 美团收银 |
|---|---|---|
| 关联实体 | `recharge_orders` | `orders` |
| 是否扣余额 | ❌（充值是加） | ❌（不走余额） |
| 是否加充值积分 | ✅ | ❌ |
| 是否加消费积分 | ❌ | ✅ |
| 是否扣库存 | ❌ | ✅ |
| 回调可能丢失 | 较少 | **较常见**（要 cron 兜底） |

### 3.3 美团 5 分钟未回调的处理

```ts
// src/payment/meituan/meituan-sync.cron.ts
@Cron('*/5 * * * *')
async syncSettling() {
  const candidates = await this.prisma.order.findMany({
    where: {
      status: 'settling',
      payMethod: 'meituan',
      paidAt: { lt: dayjs().subtract(10, 'minute').toDate() },
    },
    take: 50,
  });

  for (const order of candidates) {
    try {
      const mtStatus = await this.meituanClient.queryOrder(order.orderNo);

      if (mtStatus.status === 'PAID') {
        await this.applyMeituanCallback({
          mtTradeNo: mtStatus.mtTradeNo,
          outTradeNo: order.orderNo,
          amount: mtStatus.amount,
          paidAt: mtStatus.paidAt,
        });
        this.logger.info('cron 兜底美团回调成功', { orderNo: order.orderNo });
      } else if (mtStatus.status === 'CLOSED' || mtStatus.status === 'CANCELLED') {
        // 美团那边也没成功 → 我方关单
        await this.orderService.closeOrder(order.id, 'meituan_closed');
      } else if (mtStatus.status === 'NOT_FOUND') {
        // 美团根本没找到这单 → 数据不一致
        await this.markOutOfSync(order.id, '美团查无此单');
      }
      // PAYING / UNKNOWN → 下次再来
    } catch (e) {
      this.logger.error('美团状态查询异常', { orderNo: order.orderNo, error: e });
      await this.prisma.meituanSyncLog.create({
        data: { orderNo: order.orderNo, action: 'query', success: false, error: String(e) },
      });
    }
  }
}
```

`markOutOfSync` 写 `orders.status='out_of_sync'` 并触发**电话告警**。这个状态绝对要人工介入。

## 4. 余额支付

### 4.1 流程

```
POST /api/orders/{id}/pay
  ├─→ 校验：订单存在 / 用户拥有 / 状态 pending / 未过期
  ├─→ Idempotency-Key 检查（Redis SETNX）
  └─→ 事务（Serializable）：
        SELECT orders FOR UPDATE NOWAIT
        幂等命中（status=paid）→ 返回上次结果
        SELECT member_profiles FOR UPDATE
        UPDATE balance_cents -= amount  WHERE balance_cents >= amount
          影响行 0 → 余额不足（3001）
        UPDATE orders status='paid', paid_at=now()
        UPDATE cigars/drinks 实扣库存
        INSERT payment_records (success, channel='balance')
        INSERT balance_transactions (consume, -amount)
        INSERT point_transactions  (consume_earn, +amount/100)
        可能 INSERT level_change_logs
        发布事件 'order.paid' → 异步推送
```

### 4.2 风险点

| 风险 | 防御 |
|---|---|
| 余额扣两次 | `Idempotency-Key` Redis SETNX + DB `balance_transactions` 唯一约束 |
| 余额扣成负数 | UPDATE 带 `WHERE balance_cents >= amount` + DB CHECK 约束 |
| 库存超卖 | `WHERE stock - stock_locked >= qty` + 行锁线性化 |
| 事务里调外部 HTTP | **禁止**。推送/消息走 BullMQ 异步发布 |

## 5. 退款

### 5.1 退款决策树

```
管理员发起退款
  │
  ├─→ 校验：订单存在 / 已支付 / 退款金额 ≤ (actual_pay - refunded_amount)
  │
  ├─→ pay_method = 'balance'？
  │     ├─→ 是：直接走余额退款（同步完成）
  │     └─→ 否（wechat / meituan）：调外部 API（异步，等回调）
  │
  └─→ 退款成功后：
        UPDATE orders refunded_amount_cents += amount
        IF refunded_amount == actual_pay：status='refunded'
        IF 0 < refunded_amount < actual_pay：status='partially_refunded'
        INSERT refund_records (success)
        INSERT balance_transactions (refund, +amount)  -- 退到余额
        INSERT point_transactions  (consume_revoke, -amount/100)
        可能 INSERT level_change_logs（降级）
```

### 5.2 余额退款代码

```ts
async refundByBalance(params: {
  orderId: bigint;
  amountCents: bigint;
  reason: string;
  adminId: bigint;
  idemKey: string;
}): Promise<{ refundId: bigint }> {
  return this.prisma.$transaction(async (tx) => {
    // 1. 锁订单
    const order = await tx.$queryRaw<Order[]>`
      SELECT * FROM orders WHERE id = ${params.orderId} FOR UPDATE NOWAIT
    `;
    if (!order[0]) throw new BusinessException(4001, '订单不存在');
    if (order[0].pay_method !== 'balance') {
      throw new BusinessException(4002, '该订单未使用余额支付');
    }
    if (order[0].refunded_amount_cents + params.amountCents > order[0].actual_pay_cents) {
      throw new BusinessException(4003, '退款金额超过可退余额');
    }

    // 2. 增 refunded_amount（CHECK 约束兜底）
    await tx.$executeRaw`
      UPDATE orders
         SET refunded_amount_cents = refunded_amount_cents + ${params.amountCents},
             status = CASE
               WHEN refunded_amount_cents + ${params.amountCents} = actual_pay_cents THEN 'refunded'
               ELSE 'partially_refunded'
             END,
             updated_at = now()
       WHERE id = ${params.orderId}
    `;

    // 3. 写退款记录
    const refund = await tx.refundRecord.create({
      data: {
        orderId: params.orderId,
        amountCents: params.amountCents,
        reason: params.reason,
        channel: 'balance',
        status: 'success',
        operatorAdminId: params.adminId,
        idempotencyKey: params.idemKey,
        succeededAt: new Date(),
      },
    });

    // 4. 退款进余额（必须走 MemberService，写流水）
    await this.memberService.refundToBalance(tx, order[0].user_id, params.amountCents, {
      relatedType: 'refund',
      relatedNo: String(refund.id),
      description: `订单 ${order[0].order_no} 退款`,
      operatorAdminId: params.adminId,
    });

    // 5. 写操作日志
    await tx.operationLog.create({
      data: {
        adminId: params.adminId,
        module: 'order',
        action: 'refund',
        targetType: 'order',
        targetId: String(params.orderId),
        description: `退款 ${Number(params.amountCents) / 100} 元，原因：${params.reason}`,
        beforeData: truncateJson({ refundedBefore: order[0].refunded_amount_cents }),
        afterData:  truncateJson({ refundedAfter: order[0].refunded_amount_cents + params.amountCents }),
        level: 'warning',
      },
    });

    return { refundId: refund.id };
  }, { isolationLevel: 'Serializable', timeout: 10000 });
}
```

### 5.3 微信退款（异步）

```ts
async refundByWechat(params: { orderId: bigint; amountCents: bigint; reason: string; adminId: bigint }) {
  // 1. 先在 DB 写一条 pending refund_record
  const refund = await this.prisma.refundRecord.create({
    data: {
      orderId: params.orderId,
      amountCents: params.amountCents,
      reason: params.reason,
      channel: 'wechat',
      status: 'pending',
      operatorAdminId: params.adminId,
    },
  });

  // 2. 调微信退款 API
  try {
    const resp = await this.wechatClient.refund({
      outTradeNo: order.orderNo,
      outRefundNo: `REF${refund.id}`,
      reason: params.reason,
      amount: { refund: Number(params.amountCents), total: Number(order.actualPayCents), currency: 'CNY' },
      notifyUrl: this.config.WECHAT_REFUND_NOTIFY_URL,
    });
    await this.prisma.refundRecord.update({
      where: { id: refund.id },
      data: { externalRefundId: resp.refund_id, status: 'processing' },
    });
  } catch (e) {
    await this.prisma.refundRecord.update({
      where: { id: refund.id },
      data: { status: 'failed', failedReason: String(e) },
    });
    throw new BusinessException(6001, '微信退款失败');
  }

  return { refundId: refund.id, status: 'processing' };
}
```

退款回调处理（成功时）才会改 `orders.refunded_amount_cents`，逻辑同 §5.2 的事务。

### 5.4 退款的并发隐患

**错误做法**：
```sql
-- 计算可退金额时用 SUM
SELECT actual_pay_cents - COALESCE(SUM(amount_cents), 0)
  FROM orders o LEFT JOIN refund_records r ON r.order_id=o.id AND r.status='success'
 WHERE o.id=$1;
```

并发下：管理员 A 算「还能退 100」时，管理员 B 已写一条 success 退了 100，A 看不到 → 超额退款。

**正确做法**：
- `orders.refunded_amount_cents` 字段 + 事务内 `+= amount` + `CHECK (refunded_amount_cents <= actual_pay_cents)` 兜底
- 详见 `02_数据库设计补充.md §6`

## 6. 回调防丢失（cron 兜底）

### 6.1 哪些回调要兜底

| 回调 | 是否需 cron | 原因 |
|---|---|---|
| 微信支付（充值）成功 | ✅ | 微信偶发丢失 |
| 微信退款成功 | ✅ | 同上 |
| 美团支付成功 | ✅ | 美团回调延迟较大 |

### 6.2 微信充值兜底

```ts
@Cron('*/3 * * * *')   // 每 3 分钟
async syncWechatRecharge() {
  const candidates = await this.prisma.rechargeOrder.findMany({
    where: {
      status: 'pending',
      createdAt: { lt: dayjs().subtract(5, 'minute').toDate() },
      expireAt: { gt: new Date() },        // 未过期才查
    },
    take: 30,
  });

  for (const order of candidates) {
    try {
      const status = await this.wechatClient.queryByOutTradeNo(order.orderNo);
      if (status.trade_state === 'SUCCESS') {
        await this.rechargeService.applyWechatCallback({
          out_trade_no: order.orderNo,
          transaction_id: status.transaction_id,
          amount: status.amount,
          success_time: status.success_time,
        });
        this.logger.info('cron 兜底充值入账', { orderNo: order.orderNo });
      }
    } catch (e) {
      // 单个失败不影响其他
      this.logger.warn('查询微信状态失败', { orderNo: order.orderNo });
    }
  }
}
```

### 6.3 订单超时关单

```ts
@Cron('*/2 * * * *')   // 每 2 分钟
async closeExpiredOrders() {
  // 用 SKIP LOCKED 避免多个 worker 抢
  const expired = await this.prisma.$queryRaw<Order[]>`
    SELECT * FROM orders
     WHERE status = 'pending' AND expire_at < now()
     ORDER BY id
     LIMIT 100
     FOR UPDATE SKIP LOCKED
  `;

  for (const o of expired) {
    await this.prisma.$transaction(async (tx) => {
      // 1. 关单
      await tx.order.update({
        where: { id: o.id },
        data: { status: 'cancelled', cancelledAt: new Date(), cancelReason: 'expired' },
      });
      // 2. 回退预占库存（关键！）
      for (const item of await tx.orderItem.findMany({ where: { orderId: o.id } })) {
        if (item.itemType === 'cigar') {
          await tx.$executeRaw`
            UPDATE cigars SET stock_locked = stock_locked - ${item.qty}, updated_at=now()
             WHERE id = ${item.itemId}
          `;
        } else {
          await tx.$executeRaw`
            UPDATE drinks SET stock_locked = stock_locked - ${item.qty}, updated_at=now()
             WHERE id = ${item.itemId}
          `;
        }
      }
      // 3. payment_records 置 closed（如果有）
      await tx.paymentRecord.updateMany({
        where: { orderId: o.id, status: 'pending' },
        data: { status: 'closed' },
      });
    });
  }
}
```

> **常见错误**：忘了回退 stock_locked → 一个月后 `stock_locked` 累积，看起来"卖光了"实际卖不出。**测试用例 TC-007 必须覆盖**。

## 7. 每日对账

### 7.1 对账目标

每天凌晨 02:00 跑一次，验证三类不变量：

1. **用户资金**：`member_profiles.balance_cents == Σ(balance_transactions × direction)`
2. **订单资金**：`orders.actual_pay_cents == Σ(payment_records[success].amount_cents)`
3. **充值资金**：`recharge_orders[success].amount_cents == 对应 balance_transactions.amount_cents`
4. **微信对账**：与微信下载的对账单逐笔比对（金额、单号、状态）
5. **美团对账**：同上

### 7.2 SQL 检查

```sql
-- 检查 1：用户余额一致性
WITH bt_sum AS (
  SELECT user_id, COALESCE(SUM(amount_cents * direction), 0) AS sum_balance
    FROM balance_transactions
   GROUP BY user_id
)
SELECT u.id, mp.balance_cents, bt.sum_balance
  FROM users u
  JOIN member_profiles mp ON mp.user_id = u.id
  LEFT JOIN bt_sum bt ON bt.user_id = u.id
 WHERE mp.balance_cents <> COALESCE(bt.sum_balance, 0);
-- 返回任何行都是异常

-- 检查 2：订单实付金额一致性
SELECT o.id, o.order_no, o.actual_pay_cents,
       COALESCE(SUM(p.amount_cents), 0) AS sum_payment
  FROM orders o
  LEFT JOIN payment_records p ON p.order_id=o.id AND p.status='success'
 WHERE o.status IN ('paid','completed','partially_refunded','refunded')
 GROUP BY o.id
HAVING o.actual_pay_cents <> COALESCE(SUM(p.amount_cents), 0);

-- 检查 3：充值金额与流水
SELECT r.id, r.order_no, r.amount_cents, bt.amount_cents AS bt_amount
  FROM recharge_orders r
  LEFT JOIN balance_transactions bt
    ON bt.user_id = r.user_id
   AND bt.related_type = 'recharge'
   AND bt.related_no = r.order_no
   AND bt.type = 'recharge'
 WHERE r.status = 'success'
   AND (bt.id IS NULL OR bt.amount_cents <> r.amount_cents);
```

### 7.3 微信对账单

```ts
@Cron('0 2 * * *')   // 02:00
async reconcileWechat() {
  const date = dayjs().subtract(1, 'day').format('YYYYMMDD');

  // 1. 下载微信对账单（v3）
  const billUrl = `https://api.mch.weixin.qq.com/v3/bill/tradebill?bill_date=${dayjs().subtract(1, 'day').format('YYYY-MM-DD')}&bill_type=SUCCESS`;
  const downloadUrl = await this.wechatClient.getBillDownloadUrl(billUrl);
  const csv = await this.wechatClient.downloadBill(downloadUrl);    // 已 GZIP 解压
  const wechatRecords = this.parseWechatBill(csv);

  // 2. 拉本地数据
  const localPayments = await this.prisma.paymentRecord.findMany({
    where: {
      channel: 'wechat',
      status: 'success',
      paidAt: {
        gte: dayjs().subtract(1, 'day').startOf('day').toDate(),
        lt: dayjs().startOf('day').toDate(),
      },
    },
  });

  // 3. 比对
  const report = this.diff(wechatRecords, localPayments);

  // 4. 写报告
  await this.prisma.reconciliationReport.create({
    data: {
      reportDate: dayjs().subtract(1, 'day').toDate(),
      channel: 'wechat',
      wechatTotalCount: wechatRecords.length,
      wechatTotalAmountCents: BigInt(wechatRecords.reduce((s, r) => s + r.amountCents, 0)),
      localTotalCount: localPayments.length,
      localTotalAmountCents: localPayments.reduce((s, p) => s + p.amountCents, 0n),
      diffCount: report.diffCount,
      diffDetails: report.details,
      hasIssue: report.diffCount > 0,
    },
  });

  if (report.diffCount > 0) {
    await this.alert.fire('每日对账发现差异', report);
  }
}
```

### 7.4 告警分级

| 差异类型 | 处理 |
|---|---|
| 资金不变量失败（金额对不上） | **电话告警**（PagerDuty/电话回执），15 分钟响应 |
| 单号本地有微信无 | **企业微信通知**，工程师 1 小时内查看 |
| 微信有本地无 | **电话告警**，可能是回调丢失 + cron 也丢，需立即人工补单 |
| 状态不一致（本地 success 但微信 closed） | **电话告警**（罕见，多半本地数据被改） |

async function main() {

## 9. 安全与密钥管理

### 9.1 密钥分级

| 密钥 | 敏感度 | 存储 | 轮换周期 |
|---|---|---|---|
| 微信 API v3 密钥 | P0 | KMS（生产） / .env（dev） | 半年 |
| 微信商户私钥 (.pem) | P0 | K8s secret + KMS 加密 | 一年（证书有效期） |
| 微信 AppSecret | P0 | KMS | 半年 |
| 美团 AppSecret | P0 | KMS | 半年 |
| JWT secret | P0 | KMS + 版本字段 `JWT_SECRET_VERSION` | 三个月 |
| 数据库主密码 | P0 | KMS | 一年 |
| AES PII 加密密钥 | P0 | KMS（不可轮换，否则旧数据无法解密；用版本字段） | 不轮换 |

### 9.2 KMS 抽象

```ts
// src/infra/kms/kms.service.ts
@Injectable()
export class KmsService {
  async getApiV3Key(): Promise<Buffer> {
    if (this.config.KMS_PROVIDER === 'local') {
      return Buffer.from(this.config.WECHATPAY_API_V3_KEY, 'utf8');
    }
    // tencent KMS：拿 ciphertext，解密
    return await this.tencentKms.decrypt(this.config.WECHATPAY_API_V3_KEY_CIPHER);
  }
  // ... 其他密钥
}
```

开发用 local，生产用云 KMS。换 provider 不改业务代码。

### 9.3 不要做的事

- **禁止**：把 `apiclient_key.pem` 提交到 git。`.gitignore` 必须包含 `*.pem` `*.key`
- **禁止**：在日志打印密钥/token/手机号明文
- **禁止**：在错误信息里返回内部细节给前端（如 SQL 报错原文）
- **禁止**：在 `payment_callbacks.raw_payload` 直接存原始报文 → 必须 `maskPayload` 脱敏后再存

### 9.4 审计

每周 SRE 抽查：
- KMS 访问日志：谁解密了什么密钥
- `operation_logs` 中 module='storedvalue' / 'order' 的 refund/adjust 记录
