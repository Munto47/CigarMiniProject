### 1.1 模块职责约定

每个域目录至少包含：
- `xxx.module.ts`（NestModule）
- `xxx.controller.ts`（HTTP 入口；无业务逻辑）
- `xxx.service.ts`（业务逻辑；事务边界）
- `dto/`（参数 DTO + class-validator 校验）
- `types/`（领域 type/interface）

**禁止**：
- Controller 直接调 Prisma；
- Service 之间循环依赖（用领域事件 EventEmitter 解耦）；
- 跨域写表（如 `OrderService` 直接 UPDATE `member_profiles`，必须经 `MemberService.deductBalance(...)`）。

| 类型 | 规则 | 示例 |
|---|---|---|
| 文件 | kebab-case | `order.controller.ts`、`refund.service.ts` |

## 3. 编码模板（重要！照着写就对）

### 3.1 Controller 模板

```ts
@ApiTags('订单')
@Controller('api/orders')
@UseGuards(JwtAuthGuard)
export class OrderController {
  constructor(private readonly orderService: OrderService) {}

  @Post()
  @ApiOperation({ summary: '创建订单' })
  @HttpCode(200)
  async create(
    @CurrentUser() user: AuthUser,
    @Headers('idempotency-key') idemKey: string,
    @Body() dto: CreateOrderDto,
  ): Promise<BaseResponse<CreateOrderRespVo>> {
    if (!idemKey) throw new BusinessException(2001, '缺少 Idempotency-Key');
    const data = await this.orderService.createOrder(user.id, dto, idemKey);
    return { code: 0, message: 'success', data, timestamp: Date.now(), requestId: '...' };
  }
}
```

要点：
- Controller **只**做：参数取值、权限检查、调 Service、返回。
- 业务异常用 `BusinessException(code, message)`，全局 Filter 转换为统一响应体。
- 写操作必须接 `Idempotency-Key` 头。

### 3.2 Service 事务模板

```ts
async createOrder(userId: bigint, dto: CreateOrderDto, idemKey: string) {
  // 1. 幂等：Redis 第一道
  const cached = await this.redis.get(`idem:order:${userId}:${idemKey}`);
  if (cached) return JSON.parse(cached);

  // 2. 业务事务
  const result = await this.prisma.$transaction(async (tx) => {
    // 2a. 加锁（NOWAIT）
    const cigar = await tx.$queryRaw<Cigar[]>`
      SELECT id, stock, stock_locked, status, member_price_cents
        FROM cigars
       WHERE id = ${dto.cigarId}
       FOR UPDATE NOWAIT
    `;
    if (!cigar[0] || cigar[0].status !== 'active') {
      throw new BusinessException(3003, '商品已下架');
    }
    if (cigar[0].stock - cigar[0].stock_locked < dto.qty) {
      throw new BusinessException(3002, '库存不足');
    }

    // 2b. 锁定库存
    await tx.$executeRaw`
      UPDATE cigars
         SET stock_locked = stock_locked + ${dto.qty},
             updated_at = now()
       WHERE id = ${dto.cigarId}
         AND stock - stock_locked >= ${dto.qty}
    `;

    // 2c. 创建订单 + 明细
    const order = await tx.order.create({
      data: { /* ... idempotency_key: idemKey */ },
    });
    await tx.orderItem.createMany({ data: [...] });

    return order;
  }, { isolationLevel: 'Serializable', maxWait: 5000, timeout: 10000 });

  // 3. 幂等结果缓存
  await this.redis.setex(
    `idem:order:${userId}:${idemKey}`,
    600,
    JSON.stringify(result),
  );

  return result;
}
```

要点：
- `$transaction` 是事务边界，**不要在事务内调外部 HTTP**（微信/美团）。
- 加锁用 `FOR UPDATE NOWAIT`（关键资金路径）或 `FOR UPDATE SKIP LOCKED`（cron 批处理）。
- 隔离级别 `Serializable` 仅用于强一致场景（资金/库存），其他用默认 `ReadCommitted`。
- 事务内只做 DB 操作，外部副作用（发推送、发消息）通过 BullMQ 异步发布。

### 3.3 资金/积分/等级三联写流水模板

```ts
// 必须封装为 MemberService.deductBalance，外部调用
async deductBalance(
  tx: Prisma.TransactionClient,
  userId: bigint,
  amountCents: bigint,
  ctx: { relatedType: string; relatedNo: string; description: string },
): Promise<{ balanceAfter: bigint; pointsDelta: bigint; levelChanged: boolean }> {
  // 1. 锁会员资料
  const profile = await tx.$queryRaw<MemberProfile[]>`
    SELECT * FROM member_profiles WHERE user_id = ${userId} FOR UPDATE
  `;

  // 2. 扣余额（带余额校验）
  const updated = await tx.$executeRaw`
    UPDATE member_profiles
       SET balance_cents = balance_cents - ${amountCents},
           total_spend_cents = total_spend_cents + ${amountCents},
           consumption_points = consumption_points + ${amountCents / 100n},
           order_count = order_count + 1,
           version = version + 1,
           updated_at = now()
     WHERE user_id = ${userId} AND balance_cents >= ${amountCents}
  `;
  if (updated === 0) throw new BusinessException(3001, '余额不足');

  const after = await tx.memberProfile.findUnique({ where: { userId } });

  // 3. 写流水（唯一约束兜底重放）
  await tx.balanceTransaction.create({
    data: {
      userId,
      type: 'consume',
      direction: -1,
      amountCents,
      balanceAfterCents: after.balanceCents,
      relatedType: ctx.relatedType,
      relatedNo: ctx.relatedNo,
      description: ctx.description,
    },
  });
  await tx.pointTransaction.create({
    data: {
      userId,
      levelType: 'consumption',
      type: 'consume_earn',
      direction: 1,
      points: amountCents / 100n,
      pointsAfter: after.consumptionPoints,
      relatedType: ctx.relatedType,
      relatedNo: ctx.relatedNo,
    },
  });

  // 4. 等级判定
  const levelChanged = await this.maybeUpdateLevel(tx, userId, 'consumption', after.consumptionPoints, ctx);

  return { balanceAfter: after.balanceCents, pointsDelta: amountCents / 100n, levelChanged };
}
```

要点：
- 这个函数**不能脱离事务调用**。签名第一个参数就是 `Prisma.TransactionClient`。
- 唯一约束 `(user_id, related_type, related_no, type)` 在 DDL 已建，重放会抛 `P2002`，外层捕获转幂等成功。
- **禁止**：`UPDATE member_profiles SET balance_cents = balance_cents - 100` 不写流水。

### 3.4 第三方回调处理模板

```ts
@Post('wechat-callback')
@HttpCode(200)
async handleWechatCallback(
  @Headers() headers: Record<string, string>,
  @RawBody() raw: Buffer,        // 用 main.ts 的 rawBody 解析器
): Promise<{ code: string; message: string }> {
  // 1. 验签
  const ok = await this.wechatVerifier.verify(headers, raw);
  if (!ok) {
    await this.logCallback({ verified: false, raw });
    throw new HttpException('signature failed', 401);
  }

  // 2. 解密
  const body = JSON.parse(raw.toString());
  const decrypted = await this.wechatVerifier.decrypt(body.resource);

  // 3. 写回调日志（含 ON CONFLICT 计数）
  const externalId = decrypted.transaction_id;
  const inserted = await this.prisma.$queryRaw`
    INSERT INTO payment_callbacks (channel, external_id, related_type, related_no,
                                   raw_payload, signature, verified)
    VALUES ('wechat', ${externalId}, 'recharge', ${decrypted.out_trade_no},
            ${maskPayload(decrypted)}, ${headers['wechatpay-signature']}, TRUE)
    ON CONFLICT (channel, external_id)
      DO UPDATE SET received_count = payment_callbacks.received_count + 1
    RETURNING id, processed
  `;

  if (inserted[0].processed) {
    return { code: 'SUCCESS', message: '已处理（重放）' };
  }

  // 4. 业务事务
  await this.rechargeService.applyWechatCallback(decrypted);

  return { code: 'SUCCESS', message: '' };
}
```

要点：
- 验签失败 401，**不入业务**。
- ON CONFLICT 计数让运维能看到重放次数。
- `payment_callbacks.processed=TRUE` 是第二道幂等防线。
- 业务事务内还有 `recharge_orders FOR UPDATE` + 状态判断，第三道防线。

### 3.5 操作日志模板（管理端）

```ts
@Permissions('storedvalue:adjust')
@Post('adjust')
async adjustBalance(
  @CurrentAdmin() admin: AdminAuth,
  @Body() dto: AdjustBalanceDto,
  @Headers('idempotency-key') idemKey: string,
  @Req() req: Request,
) {
  const { userId, type, amountCents, reason } = dto;

  // 业务执行（含日志写入）
  const result = await this.storedValueService.adjustBalance({
    adminId: admin.id,
    adminName: admin.name,
    userId,
    direction: type === 'add' ? 1 : -1,
    amountCents,
    reason,
    idemKey,
    ip: req.ip,
    userAgent: req.headers['user-agent'],
  });

  return result;
}
```

`StoredValueService.adjustBalance` 内部：
```ts
return this.prisma.$transaction(async (tx) => {
  const before = await tx.memberProfile.findUnique({ where: { userId } });

  // 调资金核心
  await this.memberService.adjustBalance(tx, userId, direction, amountCents, {
    relatedType: 'manual',
    relatedNo: idemKey,
    operatorAdminId: adminId,
    description: reason,
  });

  const after = await tx.memberProfile.findUnique({ where: { userId } });

  // 写操作日志
  await tx.operationLog.create({
    data: {
      adminId, adminName,
      module: 'storedvalue',
      action: 'adjust_balance',
      targetType: 'user',
      targetId: String(userId),
      description: `${direction === 1 ? '加' : '减'} ${amountCents / 100} 元，原因：${reason}`,
      beforeData: truncateJson({ balanceCents: before.balanceCents }, 4096),
      afterData:  truncateJson({ balanceCents: after.balanceCents  }, 4096),
      level: 'warning',
      ip, userAgent,
    },
  });

  return { newBalanceCents: after.balanceCents };
});
```

要点：
- **每个写操作都要决定要不要写 operation_log**（参考 §6.6 操作日志覆盖范围）。
- 写日志必须在同事务里（一旦业务回滚，日志也回滚，避免「显示已扣但其实没扣」）。
- `beforeData/afterData` 限制 4KB（用 `truncateJson` 工具截断）。

## 4. 错误处理

### 4.1 BusinessException

```ts
// src/common/exceptions/business.exception.ts
export class BusinessException extends HttpException {
  constructor(public readonly bizCode: number, public readonly bizMessage: string) {
    super({ code: bizCode, message: bizMessage }, BusinessException.toHttpStatus(bizCode));
  }
  static toHttpStatus(code: number): number {
    if (code >= 1001 && code <= 1003) return 401;
    if (code === 1004) return 403;
    if (code === 1005 || code === 2004) return 423;
    if (code === 1006) return 426;
    if (code === 2002 || code === 4002 || code === 4005) return 409;
    if (code === 2003) return 429;
    if (code === 4003) return 410;
    if (code >= 6001 && code <= 6003) return 502;
    if (code >= 9000) return 500;
    return 422;
  }
}
```

### 4.2 全局过滤器

```ts
@Catch()
export class HttpExceptionFilter implements ExceptionFilter {
  catch(exception: any, host: ArgumentsHost) {
    const ctx = host.switchToHttp();
    const res = ctx.getResponse<Response>();
    const req = ctx.getRequest<Request>();

    // BusinessException
    if (exception instanceof BusinessException) {
      return res.status(exception.getStatus()).json({
        code: exception.bizCode,
        message: exception.bizMessage,
        data: null,
        timestamp: Date.now(),
        requestId: req.headers['x-request-id'],
      });
    }

    // Prisma 唯一约束冲突 → 幂等命中
    if (exception?.code === 'P2002') {
      this.logger.warn('唯一约束冲突（幂等命中）', exception.meta);
      return res.status(409).json({
        code: 2002,
        message: '请求重复（幂等命中）',
        ...
      });
    }

    // 兜底
    this.logger.error('Unhandled', exception);
    res.status(500).json({ code: 9001, message: 'internal error', ... });
  }
}
```

### 4.3 不要做这些事

- **禁止**：Service 里 `try/catch` 后吞掉错误返回 null（让上层捞）。
- **禁止**：用 HTTP 200 + `code:1` 表达失败（必须用 HTTP 状态码 + 业务 code 双层）。
- **禁止**：异步失败不打日志只 `console.log`。

## 5. 日志

### 5.1 三类日志区分

| 类型 | 输出 | 用途 |
|---|---|---|
| **应用日志** | stdout / 文件 | 调试、SLA、性能 |
| **操作日志** | DB `operation_logs` | 管理后台审计、合规 |
| **业务流水** | DB `*_transactions / *_logs` | 财务对账、运营报表 |

### 5.2 应用日志格式（结构化）

```ts
this.logger.info({
  requestId: req.headers['x-request-id'],
  userId: user?.id,
  action: 'create_order',
  durationMs: Date.now() - start,
  success: true,
});
```

不要：
- `console.log('xxx')`（用 `Logger`）
- 把 token / 密码 / 手机号明文打日志（必须脱敏）
- 把整个 `req.body` 打日志（限制大小）

## 7. 安全规范

### 7.1 PII / 敏感字段处理

| 字段 | 存储 | 输出 |
|---|---|---|
| 手机号 | `phone_encrypted BYTEA`（AES-256-GCM）+ `phone_mask`（138****5678） | 仅返回 mask |
| 密码 | `password_hash`（bcrypt cost 12） | 永不返回 |
| 微信 openid/unionid | 明文 | 仅内部用，不返回前端 |
| 美团 secret | KMS 加密 | 永不返回 |
| 微信支付私钥 | KMS / 文件挂载（K8s secret） | 仅启动时加载 |

### 7.2 字符串处理

- 评论、海报文字、敏感词：HTML 实体转义（`he` 库）；
- 邮件/网址输出：先 normalize 再校验白名单；
- 文件名：strip 路径分隔符、reserved chars。

### 7.3 SQL 注入

- 一律用 Prisma；
- 必须裸写 SQL 时，用 `$queryRaw` / `$executeRaw` 的标签模板（自动参数化），**禁止**字符串拼接 `${userInput}` 进 raw SQL。

### 7.4 限流

```ts
// 全局 ThrottlerModule
ThrottlerModule.forRoot([{ ttl: 60_000, limit: 60 }]);

// 严格限流
@UseGuards(ThrottlerGuard)
@Throttle({ login: { ttl: 60_000, limit: 5 } })
@Post('login') login(...) {}

@Throttle({ create: { ttl: 60_000, limit: 10 } })
@Post('orders') create(...) {}
```

### 8.2 单号生成器

```ts
// src/common/utils/id-generator.ts
// 格式：{prefix}{yyyyMMdd}{6位序列}
@Injectable()
export class IdGenerator {
  constructor(private readonly redis: RedisService) {}
  async next(prefix: 'ORD' | 'PAY' | 'R' | 'REF'): Promise<string> {
    const date = dayjs().format('YYYYMMDD');
    const key = `seq:${prefix}:${date}`;
    const seq = await this.redis.incr(key);
    await this.redis.expire(key, 86400 * 2);  // 保留 2 天
    return `${prefix}${date}${String(seq).padStart(6, '0')}`;
  }
}
```

### 8.3 脱敏工具

// src/common/utils/mask.ts
export function maskPhone(phone: string): string {
  if (!phone || phone.length !== 11) return phone;

## 9. Code Review Checklist（自查）

提交 PR 前过一遍：

### 接口层
- [ ] Controller 不写业务逻辑
- [ ] DTO 用 class-validator 装饰器
- [ ] 写操作有 `Idempotency-Key` 头并被使用
- [ ] @ApiOperation / @ApiResponse 描述齐全
- [ ] 限流装饰器到位

### 业务层
- [ ] 资金/积分/等级变更走事务
- [ ] 事务内不调外部 HTTP
- [ ] 事务隔离级别合理（资金 = Serializable）
- [ ] 加锁策略合理（NOWAIT 或 SKIP LOCKED）
- [ ] 流水唯一约束兜底
- [ ] 异常用 BusinessException

### 数据层
- [ ] 没有直接 UPDATE 业务表跳过流水
- [ ] 索引添加（高频查询字段）
- [ ] CHECK 约束保护边界
- [ ] DDL 变更有 migration 文件

### 安全
- [ ] PII 字段加密存储
- [ ] 输出脱敏
- [ ] RBAC 装饰器到位
- [ ] 操作日志写入

### 测试
- [ ] 单元测试覆盖 happy path + 1 个异常
- [ ] 接口测试 200 / 关键错误码
- [ ] 并发场景（资金/库存）有专项测试

### 日志

- [ ] 异常有 logger.error
- [ ] 不打印敏感字段

### 文档

## 10. Commit Message 规范

参考 Conventional Commits：

```
<type>(<scope>): <subject>

<body>

<footer>
```

| type | 用途 |
|---|---|
| feat | 新功能 |
| fix | 修 bug |
| refactor | 重构（无功能变更） |
| perf | 性能优化 |
| test | 仅测试改动 |
| docs | 仅文档 |
| chore | 杂项（依赖、脚本） |
| build | 构建相关 |

示例：
```
feat(order): 实现订单创建 + 库存预占 (#123)

- 新增 POST /api/orders
- 库存通过 stock_locked 字段预占
- 幂等键 idempotency_key 持久化

Closes #123
```
