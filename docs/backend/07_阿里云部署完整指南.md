```
    │ (2C 4G, 100G SSD) │  │ (1G 标准版)       │  │  (图片/文件存储)   │
    └───────────────────┘  └───────────────────┘  └───────────────────┘
```

### 1.2 域名规划

| 子域名 | 用途 | 说明 |
|---|---|---|
| `cigar.ruimacode.cn` | 后端 API | 小程序 `request.js` 生产环境指向此地址 |
| `admin.cigarpro.com` | 管理后台 | OSS 静态托管 + CDN 加速 |
| `static.cigarpro.com` | 静态资源（可选） | OSS 图片/文件 CDN 域名 |

### 1.3 与现有 Runbook 的关系

现有的 `06_部署与运维Runbook.md` 覆盖 K8s/容器化/灰度/监控等高级运维主题，本文档专注于**阿里云平台上的首次部署**，使用最精简的架构（单 ECS + 云数据库），适合项目初期。后续流量增长时可平滑迁移到 Runbook 中的多实例架构。

- 初期可用 Redis 1G 版（主从），后续扩容
- 免费 SSL 证书每个子域名各申请一个

## 3. 前置准备：域名 + SSL + 小程序配置

### 3.1 域名购买与解析

1. 登录 [阿里云域名控制台](https://dc.console.aliyun.com)
2. 购买域名 `cigarpro.com`（如已有则跳过）
3. 进入 **云解析 DNS** → 添加域名 → 添加以下记录：

| 记录类型 | 主机记录 | 记录值 | 说明 |
|---|---|---|---|
| A | `@` | `<ECS 公网 IP>` | 根域名 |
| A | `api` | `<ECS 公网 IP>` | API 后端 |
| CNAME | `admin` | `<OSS Bucket 域名>` | 管理后台（创建 OSS 后填） |
| CNAME | `static` | `<CDN 域名>` | 静态资源（可选，创建 CDN 后填） |

### 3.2 SSL 证书申请

1. 进入 [SSL 证书控制台](https://yundun.console.aliyun.com) → **免费证书**
2. 申请两个免费 DV 证书：
   - `cigar.ruimacode.cn` → 部署到 ECS Nginx
   - `admin.cigarpro.com` → 部署到 CDN（或 ECS Nginx）
3. 下载 Nginx 格式证书（`.pem` + `.key`），后续部署到 ECS

### 3.3 微信小程序后台配置

1. 登录 [微信公众平台](https://mp.weixin.qq.com) → 开发管理 → 开发设置
2. **服务器域名** → 添加 `request合法域名`：
   - `https://cigar.ruimacode.cn`
3. **uploadFile合法域名**（如小程序直接上传）：
   - `https://static.cigarpro.com`（或 OSS 直接上传域名）
4. **downloadFile合法域名**：
   - `https://static.cigarpro.com`
5. **业务域名**（如有 webview）：
   - `https://admin.cigarpro.com`

## 4. 第一步：购买与初始化云数据库 PostgreSQL

### 4.1 创建实例

1. 进入 [RDS 控制台](https://rdsnext.console.aliyun.com) → **创建实例**
2. 配置如下：

| 参数 | 值 |
|---|---|
| 数据库类型 | PostgreSQL |
| 版本 | **15**（与 `docker-compose.dev.yml` 一致） |
| 系列 | 基础版（单节点，开发/初期）或 高可用版（主从，生产推荐） |
| 规格 | 2C 4G |
| 存储 | 100G SSD 云盘 |
| 网络 | **专有网络 VPC**（与 ECS 同 VPC 同可用区） |
| 白名单 | 添加 ECS 内网 IP 段 |

3. **重要**：创建时设置 **数据库名称** 为 `cigar_db`，**账号** 为 `cigar`，**密码** 为一个强密码（记录为 `PG_PASSWORD`）

### 4.2 初始化数据库表结构

ECS 服务器就绪后，在 ECS 上执行：

```bash
# 1. 安装 Prisma CLI（在项目目录内）
cd /opt/cigarpro/Cigar_server
pnpm install --prod=false
pnpm prisma generate

# 2. 执行迁移（创建所有表）
pnpm prisma migrate deploy

# 3. 导入种子数据
pnpm db:seed
```

### 4.3 验证连接

```bash
# 在 ECS 上测试连接
psql "postgresql://cigar:<PG_PASSWORD>@<RDS内网地址>:5432/cigar_db" -c "\dt"
```

### 4.4 配置备份

RDS 控制台 → 备份恢复 → 设置自动备份策略：
- 备份周期：每天
- 备份时间：凌晨 03:00-04:00
- 日志备份：开启（WAL 归档）

## 6. 第三步：创建 OSS 对象存储 Bucket

### 6.1 创建 Bucket

1. 进入 [OSS 控制台](https://oss.console.aliyun.com) → **创建 Bucket**
2. 配置如下：

| 参数 | 值 |
|---|---|
| Bucket 名称 | `cigarpro-assets`（需全局唯一，可用 `cigarpro-assets-<随机后缀>`） |
| 地域 | 与 ECS 同地域 |
| 存储类型 | 标准存储 |
| 读写权限 | **私有** |
| 服务端加密 | 开启（AES-256） |

### 6.2 创建 AccessKey

1. 进入 [RAM 控制台](https://ram.console.aliyun.com) → 用户 → 创建用户
2. 用户名为 `cigarpro-oss`，勾选 **OpenAPI 调用访问**
3. 授权策略：`AliyunOSSFullAccess`
4. 记录 **AccessKey ID** 和 **AccessKey Secret**

### 6.3 配置 CORS（允许小程序和 Web 访问）

Bucket 列表 → 数据安全 → 跨域设置 → 创建规则：

```
来源: *
允许 Methods: GET, POST, PUT, DELETE, HEAD
允许 Headers: *
暴露 Headers: ETag
缓存时间: 600
```

### 6.4 配置 Bucket 为静态网站（管理后台部署用）

Bucket 列表 → 数据管理 → 静态页面：
- 默认首页：`index.html`
- 默认 404 页：`index.html`（SPA 路由）

## 7. 第四步：购买 ECS 云服务器并部署后端

### 7.1 创建 ECS

1. 进入 [ECS 控制台](https://ecs.console.aliyun.com) → **创建实例**
2. 配置如下：

| 参数 | 值 |
|---|---|
| 付费方式 | 包年包月（便宜）或 按量付费 |
| 地域 | 与数据库同地域同可用区 |
| 实例规格 | 2 vCPU + 4 GiB（如 ecs.c6.large） |
| 镜像 | **Ubuntu 22.04** 或 **Alibaba Cloud Linux 3** |
| 系统盘 | 40G ESSD |
| 网络 | 与数据库同 VPC 同安全组 |
| 公网 IP | **分配公网 IPv4 地址**（按量计费） |

3. **安全组规则**（入方向）：

| 端口 | 来源 | 说明 |
|---|---|---|
| 22 | 你的办公 IP | SSH 管理 |
| 80 | 0.0.0.0/0 | HTTP（重定向到 HTTPS） |
| 443 | 0.0.0.0/0 | HTTPS |

> **不要**开放 3000 端口（后端只通过 Nginx 内网代理，不走公网）

### 7.2 SSH 登录并安装基础环境

```bash
ssh root@<ECS_公网IP>

# 更新系统
apt update && apt upgrade -y

# 安装 Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 安装 pnpm
npm install -g pnpm

# 安装 PM2（进程守护）
npm install -g pm2

# 安装 Nginx
apt install -y nginx

# 安装 PostgreSQL 客户端（调试用）
apt install -y postgresql-client

# 安装 Git
apt install -y git

# 验证
node --version   # 应 >= 20
pnpm --version
pm2 --version
nginx -v
```

### 7.3 上传项目代码

**方式一：Git 拉取（推荐）**

```bash
mkdir -p /opt/cigarpro
cd /opt/cigarpro
git clone <你的私有仓库地址> .
```

### 7.4 创建生产环境配置

在 ECS 上创建 `/opt/cigarpro/Cigar_server/.env.production`：

```bash
# ============================================================
# CigarPro 生产环境配置
# ============================================================

# 应用
NODE_ENV=production
PORT=3000
API_PREFIX=api

# PostgreSQL (RDS 内网地址)

# Redis (Redis 内网地址)
REDIS_HOST=<Redis内网地址>
REDIS_PORT=6379
REDIS_PASSWORD=<REDIS_PASSWORD>

# JWT（用 openssl rand -hex 32 生成）
JWT_ACCESS_SECRET=<生成64字符随机字符串>
JWT_REFRESH_SECRET=<生成64字符随机字符串>
JWT_ACCESS_EXPIRES_ADMIN=2h
JWT_ACCESS_EXPIRES_USER=30m
JWT_REFRESH_EXPIRES=7d

# 管理员登录失败锁定

ADMIN_LOGIN_LOCK_MINUTES=15

# OSS 对象存储（替代 MinIO）
OSS_REGION=oss-cn-<region>
OSS_BUCKET=cigarpro-assets
OSS_ACCESS_KEY_ID=<RAM AccessKey ID>
OSS_ACCESS_KEY_SECRET=<RAM AccessKey Secret>
OSS_ENDPOINT=https://oss-cn-<region>.aliyuncs.com

# 微信小程序（生产环境真实密钥）
WECHAT_APP_ID=<你的小程序AppID>
WECHAT_APP_SECRET=<你的小程序AppSecret>
WECHAT_MOCK_MODE=false

# 手机号加密（必须为 32 字节的 base64 密钥）

# 微信支付 v3（生产环境真实密钥）
WECHAT_MCH_ID=<商户号>
WECHAT_API_V3_KEY=<APIv3密钥>
WECHAT_SERIAL_NO=<证书序列号>
```

## 8. 第五步：部署管理后台前端到 OSS + CDN

### 8.1 修改管理后台 API 地址

在本地项目 `Cigar_admin/` 中，修改生产环境 API 地址：

```javascript
// Cigar_admin/src/api/client.js 中的 baseURL 改为生产地址
const apiClient = axios.create({
  baseURL: 'https://cigar.ruimacode.cn/api',
  // ... 其他配置
})
```

> 如果使用 Vite proxy 模式，需要改为直接指向生产 API；或者部署时通过 Nginx 反向代理。

### 8.2 构建管理后台

```bash
# 在本地执行（Windows PowerShell）
cd D:\munto\code\WORK\CigarPro\Cigar_admin
npm install
npm run build
```

构建产物在 `Cigar_admin/dist/` 目录中。

### 8.3 上传到 OSS

**方式一：阿里云 OSS 控制台（最简单）**

直接用网页端将 `dist/` 目录所有文件上传到 Bucket 根目录。

**方式二：使用 ossutil 命令行**

```bash
# 安装 ossutil
wget https://gosspublic.alicdn.com/ossutil/1.7.19/ossutil64
chmod +x ossutil64

# 配置
./ossutil64 config --access-key-id <AccessKey ID> --access-key-secret <AccessKey Secret> --endpoint https://oss-cn-<region>.aliyuncs.com

# 上传 dist 目录
./ossutil64 cp -r ./Cigar_admin/dist/ oss://cigarpro-assets/ -u
```

### 8.4 配置 CDN 加速

1. 进入 [CDN 控制台](https://cdn.console.aliyun.com) → 添加域名
2. 配置：

| 参数 | 值 |
|---|---|
| 加速域名 | `admin.cigarpro.com` |
| 源站类型 | OSS 域名 |
| 源站地址 | `<Bucket名称>.oss-cn-<region>.aliyuncs.com` |
| 端口 | 443 |
| 加速区域 | 仅中国大陆 |

3. CDN 域名管理 → HTTPS 配置 → 上传 SSL 证书（`admin.cigarpro.com`）
4. 回源配置 → 开启 **协议跟随**
5. 缓存配置 → 自定义页面缓存规则：
   - `index.html` → 缓存 0 秒（不缓存）
   - `*.js, *.css` → 缓存 30 天
   - `*.png, *.jpg, *.svg` → 缓存 30 天

6. 回到 **云解析 DNS**，将 `admin` 的 CNAME 记录值改为 CDN 分配的 CNAME 地址

### 8.5 为 OSS 设置 CDN（图片/文件加速）

如果单独用 `static.cigarpro.com` 做图片加速：

1. CDN 添加域名 `static.cigarpro.com`，源站为同一个 OSS Bucket
2. DNS 添加 CNAME `static` → CDN CNAME
3. 后端 `.env.production` 中设置 `OSS_CDN_DOMAIN=https://static.cigarpro.com`

## 9. 第六步：Nginx 反向代理与 HTTPS 配置

### 9.1 上传 SSL 证书到 ECS

```bash
# 创建证书目录
mkdir -p /etc/nginx/ssl

# 上传证书文件（在本地 Windows 上用 scp）
scp cigar.ruimacode.cn.pem root@<ECS_IP>:/etc/nginx/ssl/
scp cigar.ruimacode.cn.key root@<ECS_IP>:/etc/nginx/ssl/
```

### 9.2 Nginx 配置文件

创建 `/etc/nginx/sites-available/cigarpro`：

```nginx
# HTTP → HTTPS 重定向
server {
    listen 80;
    server_name cigar.ruimacode.cn admin.cigarpro.com;
    return 301 https://$host$request_uri;
}

# ============================================================
# API 后端
# ============================================================
server {
    listen 443 ssl http2;
    server_name cigar.ruimacode.cn;

    ssl_certificate     /etc/nginx/ssl/cigar.ruimacode.cn.pem;
    ssl_certificate_key /etc/nginx/ssl/cigar.ruimacode.cn.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers off;

    # 上传文件大小限制
    client_max_body_size 20m;

    # 限流（API 专用）
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=30r/s;
    limit_req zone=api_limit burst=50 nodelay;

    location /api/ {
        limit_req zone=api_limit burst=50 nodelay;
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_connect_timeout 10s;
    }

    # 健康检查（不设限）
    location /api/health {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
    }

    # Swagger 文档（生产可关闭）
    location /swagger {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        # 添加基础认证（可选）
        auth_basic "Swagger Docs";
        auth_basic_user_file /etc/nginx/.htpasswd;
    }
}

# ============================================================
# 管理后台（如果不用 CDN + OSS，直接走 ECS）
# ============================================================
server {
    listen 443 ssl http2;
    server_name admin.cigarpro.com;

    ssl_certificate     /etc/nginx/ssl/admin.cigarpro.com.pem;
    ssl_certificate_key /etc/nginx/ssl/admin.cigarpro.com.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;

    root /var/www/cigarpro-admin;
    index index.html;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源长缓存
    location /assets/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

### 9.3 启用配置

```bash

ln -s /etc/nginx/sites-available/cigarpro /etc/nginx/sites-enabled/

# 测试配置
```

## 附录 B：OSS 适配说明

项目当前使用 MinIO（兼容 S3 API），迁移到阿里云 OSS 有两种方式：

### 方式一：使用阿里云 OSS SDK（推荐）

后端 `Cigar_server/src/infra/minio/` 模块需要修改，将 `@aws-sdk/client-s3` 替换为 `ali-oss`（`npm install ali-oss`），适配 OSS API。

涉及的改动点：
- `Cigar_server/src/infra/minio/minio.service.ts`
- `Cigar_server/src/upload/` 上传模块

### 方式二：OSS 使用 S3 兼容模式（最简单）

阿里云 OSS 支持 S3 兼容 API，只需修改环境变量中的 endpoint 和 access key：

```bash
# 后端直接使用 S3 兼容模式连 OSS
MINIO_ENDPOINT=https://oss-cn-<region>.aliyuncs.com
MINIO_PORT=443
MINIO_USE_SSL=true
MINIO_ACCESS_KEY=<OSS AccessKey ID>
MINIO_SECRET_KEY=<OSS AccessKey Secret>
MINIO_BUCKET=cigarpro-assets
```

这个方式**无需修改代码**，但如果原代码使用了 `ForcePathStyle`，需要确认 OSS 支持情况（OSS 默认使用 virtual-hosted-style）。

## 附录 C：费用优化建议（项目初期）

如果初期预算有限，可以进一步压缩：

1. **取消独立 RDS**：在 ECS 上用 Docker 自行运行 PostgreSQL + Redis
   - 省 ¥550/月
   - 缺点：需要自己做备份和数据安全
2. **取消 CDN**：管理后台直接从 ECS Nginx 出
   - 省 ¥15/月
3. **使用按量付费 ECS**：先跑起来，后期转包年

**最小可行部署月费：约 ¥180/月**（ECS 2C 4G + EIP + OSS + 域名）
