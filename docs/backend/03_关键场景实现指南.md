## 1. 幂等控制三件套

### 1.1 问题

用户在弱网下双击「立即支付」，2 秒内同一请求到达服务端两次。如果不防护：
- 余额扣两次
- 流水写两条
- 等级算两次（连升两级）

### 1.2 三件套

| 防线 | 位置 | 实现 |
|---|---|---|
| **客户端键** | HTTP 头 `Idempotency-Key: <UUID>` | 前端在用户首次点击时生成 UUID，多次重试用同一个 |
| **Redis SETNX** | 进程内第一道 | 命中则返回缓存的上次结果 |
| **DB 唯一约束** | 数据落地最后一道 | 即便 Redis 失效，DB 也挡得住 |

### 1.3 装饰器封装

```ts
// src/common/decorators/idempotent.decorator.ts
export const IDEMPOTENT_TTL = 'idempotent_ttl';
export const Idempotent = (ttlSeconds = 600) =>
  SetMetadata(IDEMPOTENT_TTL, ttlSeconds);

// src/common/interceptors/idempotent.interceptor.ts
@Injectable()
export class IdempotentInterceptor implements NestInterceptor {
  constructor(
    private readonly redis: RedisService,
    private readonly reflector: Reflector,
  ) {}

  async intercept(ctx: ExecutionContext, next: CallHandler): Promise<Observable<any>> {
    const ttl = this.reflector.get<number>(IDEMPOTENT_TTL, ctx.getHandler());
    if (!ttl) return next.handle();

    const req = ctx.switchToHttp().getRequest<Request>();
    const key = req.headers['idempotency-key'] as string;
    if (!key) throw new BusinessException(2001, '缺少 Idempotency-Key 请求头');
    if (!/^[0-9a-f-]{8,64}$/i.test(key))
      throw new BusinessException(2001, 'Idempotency-Key 格式非法');

    const userId = (req as any).user?.id ?? 'anonymous';
    const route = req.route?.path ?? req.url;
    const redisKey = `idem:${route}:${userId}:${key}`;

    // 抢占
    const ok = await this.redis.set(redisKey, 'pending', 'EX', ttl, 'NX');
    if (ok !== 'OK') {
      // 已存在
      const cached = await this.redis.get(redisKey);
      if (cached === 'pending') {
        throw new BusinessException(2002, '请求处理中，请稍后重试');
      }
      try {
        return of(JSON.parse(cached));
      } catch {
        throw new BusinessException(2002, '请求重复（幂等命中）');
      }
    }

    // 执行业务
    return next.handle().pipe(
      tap((data) => {
        this.redis.set(redisKey, JSON.stringify(data), 'EX', ttl);
      }),
      catchError((err) => {
        this.redis.del(redisKey);    // 失败时释放，允许客户端重试
        throw err;
      }),
    );
  }
}
```

使用：

```ts
@Post()
@Idempotent(600)
async create(...) { /* ... */ }
```

### 1.4 业务表唯一兜底

`orders.idempotency_key` UNIQUE，即便 Redis 失效，`prisma.order.create` 也会抛 `P2002`，全局 Filter 捕获后转为「幂等命中」语义。

```ts
try {
  return await this.prisma.order.create({ data: { ..., idempotencyKey: idemKey } });
} catch (e) {
  if (e.code === 'P2002' && e.meta?.target?.includes('idempotency_key')) {
    // DB 兜底命中：返回上次创建的订单
    return this.prisma.order.findFirst({ where: { userId, idempotencyKey: idemKey } });
  }
  throw e;
}
```

## 2. 事务与隔离级别

### 2.1 何时用事务

任何「跨表写」**必须**事务：
- 创建订单 → orders + order_items + cigars.stock_locked
- 支付成功 → member_profiles + balance_transactions + point_transactions + level_change_logs + payment_records + orders + cigars
- 退款 → refund_records + member_profiles + balance_transactions + point_transactions + orders.refunded_amount_cents + cigars.stock(可选)

### 2.2 事务边界

**事务边界 = Service 方法**。Controller 不参与事务；Repository 也不参与（只是封装 SQL）。

```ts
// ✅ 正确
async pay(orderId, userId) {
  return this.prisma.$transaction(async (tx) => {
    // 1. 锁订单
    // 2. 扣余额
    // 3. 写流水
    // 4. 标支付
  }, { isolationLevel: 'Serializable' });
}

// ❌ 错误：跨事务
async pay(orderId, userId) {
  await this.prisma.order.update({ where:{id:orderId}, data:{ status:'paid' } });
  await this.prisma.balanceTransaction.create({ data:... });    // 如果这里失败，订单已 paid 但流水缺失
}
```

### 2.3 隔离级别选择

| 隔离级别 | 用法 | 性能 |
|---|---|---|
| `ReadCommitted`（默认） | 读多写少，无金额 | 最快 |
| `RepeatableRead` | 报表/统计（避免幻读） | 中等 |
| `Serializable` | 资金/库存/积分 | 慢（可能抛序列化失败需重试） |

```ts
prisma.$transaction(fn, { isolationLevel: 'Serializable' });
```

### 2.4 序列化失败重试

`Serializable` 下 PG 会抛 `40001 serialization_failure`。**必须重试**：

```ts
// src/common/utils/retry.ts
export async function withSerializableRetry<T>(
  fn: () => Promise<T>,
  maxAttempts = 3,
): Promise<T> {
  for (let i = 0; i < maxAttempts; i++) {
    try {
      return await fn();
    } catch (e: any) {
      if (e.code === 'P2034' || e.message?.includes('40001')) {
        if (i < maxAttempts - 1) {
          await new Promise((r) => setTimeout(r, 50 * Math.pow(2, i)));
          continue;
        }
      }
      throw e;
    }
  }
  throw new Error('unreachable');
}

// 用法：
await withSerializableRetry(() =>
  this.prisma.$transaction(async (tx) => { /* ... */ }, { isolationLevel: 'Serializable' })
);
```

### 2.5 事务内禁忌

| 禁忌 | 原因 |
|---|---|
| 调外部 HTTP（微信/美团） | 网络慢，事务时间过长，锁占用，连接耗尽 |
| 发邮件、推送、Slack | 同上 |
| 发 BullMQ 队列消息 | BullMQ 不在事务内，事务回滚但消息已发出 → 不一致 |
| 长循环（>500ms） | 锁占用 |

**正确做法**：事务内只更新数据库；外部副作用通过 `EventEmitter` 在事务提交后触发：

```ts
async pay(orderId, userId) {
  const result = await this.prisma.$transaction(async (tx) => { /* ... */ });

  // 事务已提交，再发推送
  this.eventEmitter.emit('order.paid', { orderId, userId });
  return result;
}

@OnEvent('order.paid')
async handleOrderPaid(payload) {
  await this.queue.add('push-notification', payload);
  await this.queue.add('meituan-sync',     payload);
}
```

## 3. 行级锁与并发控制

### 3.1 三种锁姿势

| 姿势 | SQL | 何时用 |
|---|---|---|
| 悲观锁默认 | `FOR UPDATE` | 读写都串行（慢） |
| 拿不到立即失败 | `FOR UPDATE NOWAIT` | 用户支付路径（不让用户等） |
| 拿不到跳过 | `FOR UPDATE SKIP LOCKED` | cron 批处理 |

### 3.2 用户支付：NOWAIT

```ts
const order = await tx.$queryRaw<OrderRow[]>`
  SELECT id, status, actual_pay_cents
    FROM orders
   WHERE id = ${orderId}
   FOR UPDATE NOWAIT
`;
// 拿不到锁 → PG 抛 55P03 lock_not_available → 转 BusinessException 2004
```

错误处理：

```ts
try {
  await tx.$queryRaw`SELECT ... FOR UPDATE NOWAIT`;
} catch (e: any) {
  if (e.code === 'P2010' && e.meta?.code === '55P03') {
    throw new BusinessException(2004, '操作冲突，请稍后重试');
  }
  throw e;
}
```

### 3.3 cron 超时关单：SKIP LOCKED

```sql
WITH expired AS (
  SELECT id FROM orders
   WHERE status = 'pending'
     AND expire_at < now() - INTERVAL '30 seconds'
   FOR UPDATE SKIP LOCKED
   LIMIT 200
)
UPDATE orders o
   SET status = 'cancelled', cancelled_at = now(), cancel_reason = '超时未支付'
  FROM expired e
 WHERE o.id = e.id
RETURNING o.id;
```

要点：
- `SKIP LOCKED`：跳过被支付路径锁住的订单（让支付完成）
- `now() - INTERVAL '30 seconds'`：30 秒安全边界，避免 cron 与刚开始的支付互锁
- `LIMIT 200`：分批处理，避免单个事务过大

### 3.4 幂等命中分支（关键！）

```ts
// 错误：用 status 过滤
const order = await tx.$queryRaw`
  SELECT * FROM orders WHERE id=${orderId} AND status='pending' FOR UPDATE NOWAIT
`;
if (!order[0]) throw new BusinessException(4001, '订单状态不允许');
// ❌ 已支付的订单被 SELECT 过滤掉了，前端收到错误，但订单已 paid

// 正确：先取出，按状态分支
const order = await tx.$queryRaw<OrderRow[]>`
  SELECT id, status, actual_pay_cents
    FROM orders WHERE id=${orderId} FOR UPDATE NOWAIT
`;
if (!order[0]) throw new BusinessException(4001, '订单不存在');
switch (order[0].status) {
  case 'paid':
    // ✅ 幂等命中：返回上次结果
    const lastBalance = await tx.balanceTransaction.findFirst({
      where: { userId, relatedNo: orderNo, type: 'consume' },
      orderBy: { id: 'desc' },
    });
    return { idempotent: true, balanceAfterCents: lastBalance.balanceAfterCents };
  case 'cancelled':
  case 'refunded':
    throw new BusinessException(4001, '订单状态不允许此操作');
  case 'pending':
    // 走下方扣款逻辑
    break;
  default:
    throw new BusinessException(4001, `订单状态 ${order[0].status} 异常`);
}
```

## 4. 库存预占的完整生命周期

### 4.1 三阶段流转

```
[用户加购车]   不动库存
      ↓
[创建订单]     stock_locked += qty   （10 库存 → 9 可售）
      ↓
[支付成功]     stock -= qty, stock_locked -= qty   （净变化：可售 -1）
      ↓
[退款]         stock += qty   （可选，看是否回库存）
```

或者：
```
[创建订单]     stock_locked += qty   （10 库存 → 9 可售）
      ↓
[超时/取消]    stock_locked -= qty   （恢复 10 可售）
```

### 4.2 完整代码

```ts
// src/order/order.service.ts
async createOrder(userId: bigint, dto: CreateOrderDto, idemKey: string) {
  return this.prisma.$transaction(async (tx) => {
    // 1. 计算价格 snapshot + 校验商品
    const itemsWithSnapshot = await this.computeItemsSnapshot(tx, dto.items);

    // 2. 库存预占（每个 SKU 加锁 + 扣减）
    for (const item of itemsWithSnapshot) {
      const updated = await tx.$executeRaw`
        UPDATE ${item.productType === 'cigar' ? 'cigars' : 'drinks'}
           SET stock_locked = stock_locked + ${item.qty},
               updated_at = now()
         WHERE id = ${item.productId}
           AND status = 'active'
           AND stock - stock_locked >= ${item.qty}
      `;
      if (updated === 0) throw new BusinessException(3002, `${item.nameSnapshot} 库存不足`);
    }

    // 3. 创建订单
    const orderNo = await this.idGen.next('ORD');
    const order = await tx.order.create({
      data: {
        orderNo,
        idempotencyKey: idemKey,
        userId,
        userNameSnapshot: '...',
        totalCents:       sum(itemsWithSnapshot, 'priceCents'),
        memberDiscountCents: ...,
        actualPayCents:   sum(itemsWithSnapshot, 'memberPriceCents'),
        payMethod:        dto.payMethod,
        status:           'pending',
        meituanSyncStatus: dto.payMethod === 'meituan' ? 'not_synced' : 'not_required',
        expireAt:         new Date(Date.now() + 30 * 60 * 1000),
        items: {
          create: itemsWithSnapshot.map((item) => ({
            productType:        item.productType,
            productId:          item.productId,
            nameSnapshot:       item.nameSnapshot,
            specSnapshot:       item.specSnapshot,
            priceCentsSnapshot: item.priceCents,
            memberPriceSnapshot: item.memberPriceCents,
            qty:                item.qty,
            actualAmountCents:  item.memberPriceCents * BigInt(item.qty),
            thumbUrlSnapshot:   item.thumbUrl,
          })),
        },
      },
      include: { items: true },
    });

    return order;
  }, { isolationLevel: 'Serializable' });
}
```

### 4.3 支付成功时的「实扣」

写在 `PaymentService.payByBalance` 的事务内：

```ts
for (const item of order.items) {
  const table = item.productType === 'cigar' ? 'cigars' : 'drinks';

    UPDATE ${Prisma.raw(table)}
```

### 4.4 取消/超时时的回退

```ts
// src/order/order.service.ts
async cancel(orderId: bigint) {
  return this.prisma.$transaction(async (tx) => {
    // 锁订单
    const order = await tx.$queryRaw<Order[]>`
      SELECT * FROM orders WHERE id=${orderId} FOR UPDATE
    `;
    if (order[0].status !== 'pending') throw new BusinessException(4001, '订单状态不允许取消');

    // 回退库存
    const items = await tx.orderItem.findMany({ where: { orderId } });
    for (const item of items) {
      const table = item.productType === 'cigar' ? 'cigars' : 'drinks';
      await tx.$executeRaw`
        UPDATE ${Prisma.raw(table)}
           SET stock_locked = stock_locked - ${item.qty},
               updated_at   = now()
         WHERE id = ${item.productId}
      `;
    }

    // 关闭订单
    await tx.order.update({
      where: { id: orderId },
      data: { status: 'cancelled', cancelledAt: new Date(), cancelReason: '用户取消' },
    });

    // 关闭尚 pending 的支付单
    await tx.paymentRecord.updateMany({
      where: { orderId, status: 'pending' },
      data:  { status: 'closed' },
    });
  });
}
```

### 4.5 cron 超时关单（含库存回退）

```ts
// src/order/order.cron.ts
@Cron(CronExpression.EVERY_MINUTE)
async expireOverdueOrders() {
  const expired = await this.prisma.$queryRaw<{ id: bigint }[]>`
    WITH e AS (
      SELECT id FROM orders
       WHERE status='pending' AND expire_at < now() - INTERVAL '30 seconds'
       FOR UPDATE SKIP LOCKED
       LIMIT 200
    )
    SELECT id FROM e
  `;

  for (const { id } of expired) {
    try {
      await this.cancel(id);    // 复用同样的事务逻辑
    } catch (e) {
      this.logger.error(`超时关单失败 orderId=${id}`, e);
    }
  }

  if (expired.length > 0) this.logger.info(`超时关单 ${expired.length} 笔`);
}
```

## 5. 等级计算与变更

### 5.1 业务规则

- **充值等级**（recharge）：基于 `recharge_points`（=累计充值元数）。
- **消费等级**（consumption）：基于 `consumption_points`（=累计消费元数）。
- 区间在 `level_configs` 表配置（V1~V9）。
- 升级触发：充值入账后、消费入账后、退款入账后（可能降级）、管理员重算。

### 5.2 核心逻辑

```ts
// src/member/level.service.ts
@Injectable()
export class LevelService {
  /**
   * 重新评估某个用户的某种等级。
   * 必须在事务内调用，且已持有 member_profiles 锁。
   *
   * @return { changed, levelBefore, levelAfter }
   */
  async maybeUpdateLevel(
    tx: Prisma.TransactionClient,
    userId: bigint,
    levelType: 'recharge' | 'consumption',
    newPoints: bigint,
    triggerCtx: { triggerType: string; relatedNo?: string; remark?: string },
  ): Promise<LevelTransitionResult> {
    // 1. 当前等级
    const profile = await tx.memberProfile.findUnique({ where: { userId } });
    const before = levelType === 'recharge' ? profile.rechargeLevel : profile.consumptionLevel;

    // 2. 查 level_configs 找新等级
    const config = await tx.levelConfig.findFirst({
      where: {
        levelType,
        minPoints: { lte: newPoints },
        OR: [{ maxPoints: null }, { maxPoints: { gte: newPoints } }],
        enabled: true,
      },
      orderBy: { level: 'desc' },
    });
    if (!config) throw new BusinessException(9001, `等级配置缺失 type=${levelType} points=${newPoints}`);

    const after = config.level;
    if (after === before) return { changed: false, levelBefore: before, levelAfter: after };

    // 3. 更新 member_profiles
    await tx.memberProfile.update({
      where: { userId },
      data: levelType === 'recharge'
        ? { rechargeLevel:    after, version: { increment: 1 } }
        : { consumptionLevel: after, version: { increment: 1 } },
    });

    // 4. 写 level_change_logs
    await tx.levelChangeLog.create({
      data: {
        userId,
        levelType,
        levelBefore: before,
        levelAfter:  after,
        triggerType: triggerCtx.triggerType,
        relatedNo:   triggerCtx.relatedNo,
        remark:      triggerCtx.remark,
      },
    });

    return { changed: true, levelBefore: before, levelAfter: after };
  }
}
```

```ts
async recalculateAll(triggeredBy: bigint) {

    data: { triggeredBy, status: 'pending' },

// Worker
@Processor('level-recalc')
async process(job: Job<{ jobId: bigint }>) {
  await this.prisma.levelRecalcJob.update({
    where: { id: job.data.jobId },
    data: { status: 'running', startedAt: new Date() },
  });

  let affected = 0;
  let total = 0;
  const pageSize = 1000;
  let cursor: bigint | null = null;

  while (true) {
    const profiles = await this.prisma.memberProfile.findMany({
      take: pageSize,
      ...(cursor ? { cursor: { userId: cursor }, skip: 1 } : {}),
      orderBy: { userId: 'asc' },
    });
    if (profiles.length === 0) break;
    total += profiles.length;

    for (const p of profiles) {
      // 每个用户单独事务，避免大事务
      await this.prisma.$transaction(async (tx) => {
        const r1 = await this.levelService.maybeUpdateLevel(tx, p.userId, 'recharge', p.rechargePoints, {
          triggerType: 'recalculate',
        });
        const r2 = await this.levelService.maybeUpdateLevel(tx, p.userId, 'consumption', p.consumptionPoints, {
          triggerType: 'recalculate',
        });
        if (r1.changed || r2.changed) affected++;
      });
    }

    cursor = profiles[profiles.length - 1].userId;
  }

  await this.prisma.levelRecalcJob.update({
    where: { id: job.data.jobId },
    data: { status: 'success', totalUsers: total, affectedUsers: affected, finishedAt: new Date() },
  });
}
```

## 6. AI 推荐规则引擎

> **一期 P0 范围**：基于 `cigar_tags + flavor_tags.score_map` 的简单规则评分。**不**依赖外部 AI 服务。
>
> 二期可升级为向量召回 + LLM rerank。

### 6.1 输入输出

请求：
```json
POST /api/recommend
{
  "answers": { "1": [0], "2": [2], "3": [1, 3, 5], "4": [2], "5": [3] }
}
```

响应：
```json
{
  "code": 0,
  "data": {
    "cigars": [
      { "id": 12, "name": "...", "score": 8.42, "matchedTags": ["木香","烟草"] },
      ...
    ]
  }
}
```

### 6.2 算法

```
Step 1: 把用户答案累加成 user_score: { 风味维度 → 偏好分 }
        例：{ "果香": 5, "木香": 7, "烟草": 6, ... }

Step 2: 对每条在售 cigar，从 cigar_tags JOIN flavor_tags 取出标签集合
        每个标签的 score_map 累加到 cigar_score

Step 3: 计算相似度（余弦或加权命中数）
        score(cigar) = Σ user_score[k] × cigar_score[k] × tag.ai_weight
        归一化：除以 sqrt(Σ user_score² × Σ cigar_score²)

Step 4: 按 score DESC 排序，取 Top 6
```

### 6.3 实现

```ts
// src/recommend/recommend.service.ts
@Injectable()
export class RecommendService {
  async recommend(userId: bigint | null, answers: Record<string, number[]>) {
    const start = Date.now();

    // 1. 累加用户偏好
    const questions = await this.prisma.recommendQuestion.findMany({ where: { enabled: true } });
    const userScore: Record<string, number> = {};
    for (const q of questions) {
      const picked = answers[q.id.toString()] ?? answers[q.position.toString()] ?? [];
      const opts = q.options as Array<{ label: string; score_delta: Record<string, number> }>;
      for (const idx of picked) {
        const opt = opts[idx];
        if (!opt) continue;
        for (const [k, v] of Object.entries(opt.score_delta)) {
          userScore[k] = (userScore[k] ?? 0) + v;
        }
      }
    }

    // 2. 拉所有在售雪茄 + 标签
    const cigars = await this.prisma.$queryRaw<CigarWithTags[]>`
      SELECT c.id, c.name, c.brand, c.thumb_url, c.member_price_cents, c.flavor_scores,
             COALESCE(json_agg(json_build_object(
               'name', ft.name, 'category', ft.category,
               'aiWeight', ft.ai_weight, 'scoreMap', ft.score_map
             )) FILTER (WHERE ft.id IS NOT NULL), '[]') AS tags
        FROM cigars c
        LEFT JOIN cigar_tags ct ON ct.cigar_id = c.id
        LEFT JOIN flavor_tags ft ON ft.id = ct.tag_id AND ft.enabled = TRUE
       WHERE c.status = 'active' AND c.deleted_at IS NULL
       GROUP BY c.id
    `;

    // 3. 对每个雪茄计算 score
    const scored = cigars.map((c) => {
      let score = 0;
      const matched: string[] = [];
      for (const tag of c.tags) {
        for (const [k, v] of Object.entries(tag.scoreMap as Record<string, number>)) {
          if (userScore[k]) {
            score += userScore[k] * v * Number(tag.aiWeight);
            if (!matched.includes(tag.name)) matched.push(tag.name);
          }
        }
      }
      // flavor_scores 也参与（如果用户没勾选标签时仍能推荐）
      const flavorScores = c.flavor_scores as Record<string, number>;
      for (const [k, v] of Object.entries(flavorScores ?? {})) {
        if (userScore[k]) score += userScore[k] * v * 0.3;       // 较低权重
      }
      return { id: c.id, name: c.name, score, matchedTags: matched, ...c };
    });

    // 4. 排序取前 6
    scored.sort((a, b) => b.score - a.score);
    const top = scored.slice(0, 6);

    // 5. 写日志
    await this.prisma.recommendLog.create({
      data: {
        userId: userId,
        answers: answers,
        resultCigars: top.map((t) => ({ id: t.id, score: t.score })),
        costMs: Date.now() - start,
      },
    });

    return { cigars: top };
  }
}
```
